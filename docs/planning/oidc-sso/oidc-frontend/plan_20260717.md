## Global constraints (hand to every implementer + reviewer, verbatim)

- **pnpm workspace** — node_modules already installed in the worktree. Run `npx vitest run <path>` and
  `npx eslint <path>`. Do NOT run repo-wide `npm run lint` (≈34 pre-existing unrelated errors); gate is
  **no NEW eslint errors on touched files**.
- **Additive only** — do not change the password or Google sign-in behaviour. Reuse the conditional-mount
  idiom (`GoogleSignInButton.tsx:98`, `login/page.tsx:317`).
- **Runtime, not build-time** — SSO enabled-state comes from `GET /api/v1/auth/oidc/status`, never a
  `NEXT_PUBLIC_*` var (those are build-time inlined).
- **The SSO button and `/start` are a full-page navigation** (redirect flow), not an axios fetch.
- **Never render the client secret.** Show `secret_hint` for a stored secret; the secret field is
  write-only.
- **`/login/callback` MUST be added to `AuthContext.tsx` `publicRoutes`** (`:28`) or the provider bounces
  it before the token is stored.
- Theme: CSS variables only, no hardcoded colors (repo rule). Follow shadcn/settings patterns.

## Tasks (sequential)

### Task 1 — API client module `lib/api/oidc.ts`

- **File:** `services/frontend-web/lib/api/oidc.ts` **(new)** — using the shared `apiClient`
  (`lib/api-client.ts`):
  - `getOidcStatus(): Promise<{enabled: boolean; button_label: string}>` → `GET /api/v1/auth/oidc/status`
  - `getOidcConfig()` → `GET /api/v1/settings/oidc` (returns the config sans secret, `configured` flag)
  - `putOidcConfig(payload)` → `PUT /api/v1/settings/oidc`
  - `deleteOidcConfig()` → `DELETE /api/v1/settings/oidc`
  - Export TS types mirroring the backend Pydantic response (no `client_secret` in the response type).
- **TDD:** `lib/api/__tests__/oidc.test.ts` — mock `apiClient` (as `api-client.test.ts` does), assert each
  fn hits the right URL/method and returns the parsed body.
- **Commit:** `feat(oidc-sso): frontend OIDC API client module (M11)`

### Task 2 — `/login/callback` page + publicRoutes

- **Files:**

- **TDD:** `app/login/callback/__tests__/page.test.tsx` — token in hash → `login` called + `router.replace`
  to `/dashboard` (mock `useAuth`, `next/navigation`, set `window.location.hash`); no token → error state,
  no dashboard redirect. Plus an AuthContext assertion that `publicRoutes` includes `/login/callback`
  (extend the existing AuthContext test or add a tiny one).
- **Commit:** `feat(oidc-sso): /login/callback token handoff + publicRoutes (M11)`

## Edge cases & error handling

- Hash parsing: tolerate `#token=…` with or without other fragment params; never log the token.
- If `getOidcStatus()` fails (backend down / older backend), the button simply doesn't render — login page
  must still work (additive, fail-open to hidden).
- Settings: deleting when no config exists (404) → friendly "nothing to remove", not a crash.
- `allowed_email_domains` empty in the editor → the UI should warn "empty = deny-all (no one can sign in)"
  so an operator doesn't enable SSO with an empty allowlist by accident (matches backend M12 semantics).
