# Implementation Plan — `oidc-config`

**Feature:** `oidc-sso` · **Aspect:** `oidc-config` · **Date:** 2026-07-17
**Spec:** `./spec.md` · **PRD:** `../prd.md` (M2/M3/M4/M12) · **Branch:** `feat/oidc-sso`
**Executed under:** strict TDD + agents-team (one implementer per task, sequential).

> Data foundation for OIDC: the `oidc_config` table + admin CRUD + public status probe. No auth flow.

## Global constraints (hand to every implementer + reviewer, verbatim)

- **Mirror `ZendeskIntegration`** (`src/models/zendesk_integration.py`) and its route
  (`src/api/routes/zendesk_integration.py`) — same Fernet pattern, same `token_hint`/`secret_hint`
  shape, same per-org uniqueness idiom.
- **Encrypt in the ROUTE layer, never the model.** Use `src/utils/encryption.py`
  (`encrypt_api_key`/`decrypt_api_key`/`get_key_hint`).
- **Missing `LLM_ENCRYPTION_KEY` (ValueError) → HTTP 422, never 500.**
- **`client_secret` is NEVER returned** in any response — `secret_hint` only.
- **All `/settings/oidc` verbs gated `require_admin_or_owner`**, scoped by `organization_id` from the
  authenticated context.
- **Empty/absent `allowed_email_domains` = deny-all — store as-is, do NOT coerce to allow-all.**
- **Migration chains off head `b8c9d0e1f2a3`.** Verify on **scratch Postgres** via `createdb` +
  `DATABASE_URL=...` — NEVER `-x sqlalchemy.url`, NEVER the dev DB `customer_feedback_saas`.
- Backend routes here are **sync `def`** (match the integration routes), Pydantic request/response models.
- Scope test runs; never bare full-suite (pre-existing `test_report_ws.py` segfault).

## Setup

- Service: `services/backend-api` only. New files + one migration + `main.py` edits.
- No new dependencies (Fernet, FastAPI, SQLAlchemy, python-jose all present).
- Test DB: existing SQLite `conftest.py` harness. Migration verification: scratch Postgres.

## Edge cases & error handling

- `issuer_url` stored as an opaque string — **not fetched or validated here** (SSRF gate is
  `oidc-login-flow`). Do not add a network call.
- Deleting a non-existent config → 404 (match integration-route conventions).
- `allowed_email_domains` with a bad entry (empty string, whitespace, no dot) → 422 with a clear
  message; an empty **list** is valid and means deny-all.
- Status endpoint must never 500 when unconfigured — return `enabled:false`.
