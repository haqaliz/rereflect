## Global constraints (hand to every implementer + reviewer, verbatim)

- **No prior session in this flow.** The signed `state` carries ONLY a CSRF nonce hash + PKCE binding тАФ
  NEVER a user/org id. Identity comes from the validated ID token.
- **Mirror Salesforce's state/nonce mechanics** (`salesforce_integration.py`): `_hash_nonce` (SHA-256),
  `_sign_state`/`_verify_state` (HMAC keyed on `JWT_SECRET` via `from src.api.auth import JWT_SECRET`),
  `hmac.compare_digest`, HttpOnly+Secure nonce cookie scoped to the callback path, `RedirectResponse`.
- **Use `authlib` for ID-token validation** (signature vs JWKS, `iss`, `aud`, `exp`, `nonce`). Do NOT
  hand-roll JWKS parsing or signature checks.
- **SSRF gate on the issuer host BEFORE any fetch.** New shared `src/utils/ssrf.py`. Do NOT touch the
  jira/zendesk copies.
- **`email_verified: true` REQUIRED** to link or provision (M9/D6) тЖТ else 403.
- **M12 domain allowlist: empty list = DENY-ALL.** Domain not in list тЖТ 403, no user created.
- **JIT default `role="member"`**, org = the enabled config's org, `auth_provider="oidc"`.
- **Redirect target is the fixed configured `FRONTEND_URL`** тАФ never request/IdP-derived (open-redirect).
  Success тЖТ `{FRONTEND_URL}/login/callback#token=<jwt>` (FRAGMENT, not query).
- **Additive only** тАФ do not modify existing `auth.py` endpoints; only append. JWT claim shape stays
  `{user_id, organization_id, role}` via `create_access_token`.
- Migrations chain off head `c9d0e1f2a3b4`; verify on **scratch Postgres** (`createdb`+`DATABASE_URL=`),
  NEVER `-x sqlalchemy.url`, NEVER the dev DB. Scope test runs (pre-existing `test_report_ws.py` segfault).

### Task 2 тАФ OIDC provider service (discovery + JWKS + SSRF + ID-token validation)

- **Files:**
  - `src/utils/ssrf.py` **(new)** тАФ `assert_host_not_ssrf(host: str) -> None`, lifted verbatim-semantics
    from `jira_integration.py:195-223` (resolve via `socket.getaddrinfo`, reject loopback/private/
    link-local, raise a `ValueError` or a small custom exception тАФ NOT an HTTPException, so it's reusable
    off a route; the caller maps to the right response). Unit tests `tests/test_ssrf.py` (public host
    passes; `127.0.0.1`, `10.x`, `169.254.x`, unresolvable each reject).
  - `src/services/oidc_provider.py` **(new)** тАФ an `OidcProvider` built from an `OidcConfig` row:
    - `discover()` тЖТ fetch+cache `{issuer}/.well-known/openid-configuration` (TTL, e.g. 3600s; cache
      keyed by issuer). SSRF-gate the issuer host first.
    - `jwks()` тЖТ fetch+cache the `jwks_uri` keys (TTL). SSRF-gate that host too.
    - `authorization_url(state, nonce, code_challenge, redirect_uri)` тЖТ build the authorize URL.
    - `exchange_code(code, code_verifier, redirect_uri)` тЖТ POST the token endpoint, return tokens.
    - `validate_id_token(id_token, nonce)` тЖТ via `authlib` (JsonWebToken / jwt.decode with claims
      options): verify signature vs `jwks()`, `iss`==config.issuer, `aud`==config.client_id, exp/nbf,
      and `nonce`==passed nonce. Return validated claims (must include `sub`, `email`, `email_verified`).
    - All outbound HTTP via `httpx` (sync `httpx.Client`, matching the integrations' style).
  - **Tests `tests/test_oidc_provider.py`** тАФ mock `httpx` (discovery, jwks, token). Cover AC2 (SSRF),
    AC3 (wrong iss/aud/expired/bad-sig/bad-nonce reject; valid passes), AC12 (fetch-once-per-TTL: assert
    the mocked transport call count across two calls). Use a locally-generated RSA key to sign a test
    JWT and publish its public JWK in the mocked JWKS.
- **Validation:** `pytest tests/test_ssrf.py tests/test_oidc_provider.py -v` green.
- **Commit:** `feat(oidc-sso): OIDC provider service тАФ discovery/JWKS/SSRF/id_token validation (M6/M10)`

## Edge cases & error handling

- IdP returns `error=` on the callback (user denied) тЖТ generic error redirect, no user.
- Token endpoint 4xx/5xx тЖТ generic error redirect; never expose the body.
- `sub` present but no `email` claim тЖТ 403 (can't provision/enforce allowlist without email).
- Email case: lowercase before allowlist + user lookup (Google path lowercases at `google_auth.py:98`).
- A user matched by `oidc_sub` whose email now collides with a DIFFERENT existing user тЖТ do NOT merge;
  prefer the `oidc_sub` match, do not reassign email. Add a test if time permits; note otherwise.
- Discovery/JWKS host unreachable тЖТ generic error redirect; do not 500 the callback.
