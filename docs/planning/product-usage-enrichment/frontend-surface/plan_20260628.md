## Testing strategy

- Vitest: API client (mock axios), ComponentProgressBars (5 bars), UsageTimeline (series+toggle), profile card (empty + populated).
- Manual: `./start-all.sh`; POST a usage event (curl); open `/customers/{email}` → Usage card + 5th bar; raise usage weight in settings → score moves.
- `cd services/frontend-web && npm run test && npm run lint && npm run build`.

## Edge cases & errors

- No usage (404 from getUsage) → neutral empty state, no error toast.
- `usage_component` missing on older payloads → default render (treat as 50/neutral) without crashing.
- Loading via Suspense/skeleton; `useSearchParams` wrapped if used.
- Never hardcode colors (CSS vars); strict TS.
