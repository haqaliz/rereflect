## Phase 1 — Pydantic schemas + bounds (RED)

- **Goal:** validate the normalized batch body and enforce limits.
- **Files:** `services/backend-api/src/api/schemas/usage.py` (new) or inline in the route module; `tests/test_usage_ingest.py` (new).
- **Steps:**
  1. `UsageEventIn`: `type: Literal["identify","track"]`, `email: str|None`, `userId: str|None`, `event: str|None`, `name: str|None`, `timestamp: datetime|None`, `messageId: str|None`, `properties: dict = {}`, `traits: dict = {}`.
  2. `UsageBatchIn`: `events: list[UsageEventIn]` with `max_items=1000` (Pydantic `Field(max_length=1000)`); request exceeding → handled as `413` in the route (or validation→ map to 413).
  3. Email resolution helper: `event.email or event.traits.get("email")`.
  4. `properties` size guard helper (serialize, if >16 KB truncate + flag).
- **Tests (RED first):** schema accepts valid event; rejects unknown `type`; resolves email from traits; flags no-email.
- **Validation:** `pytest tests/test_usage_ingest.py -v`.
- **Commit:** `test(usage): ingest schema + bounds (RED)`

## Phase 3 — Route + auth + dedup + enqueue (GREEN)

- **Files:** `services/backend-api/src/api/routes/usage_webhooks.py` (new); register in `src/api/main.py` (`include_router`).
- **Steps:**
  1. `router = APIRouter(prefix="/api/v1/webhooks/usage", tags=["usage-webhooks"])`.
  2. `POST ""` with `auth: ApiKeyAuth = Depends(verify_api_key)` + `dependencies=[Depends(require_scope("ingest"))]`.
  3. Reject `len(events) > 1000` → `HTTPException(413)`.
  4. Per event: resolve email (none → `skipped_reasons["no_email"]++`); require `messageId` (none → `skipped_reasons["no_message_id"]++`, or hash fallback per spec); quick dedup check `usage_event` by `(org, external_event_id)` (dup → `skipped_reasons["duplicate"]++`); else write `usage_event` row scoped to `auth.organization_id` and `app.send_task("src.tasks.usage_metrics.process_usage_event", args=[org_id, email, event_type, event_name, occurred_at_iso, external_event_id, properties])`.
  5. Return `202 {accepted, skipped, skipped_reasons}`.
- **Tests:** acceptance criteria 1-7 from spec (use a mocked celery `send_task`; real DB session via test fixture).
- **Validation:** `pytest tests/ -v` green.
- **Commit:** `feat(usage): POST /api/v1/webhooks/usage ingest receiver (ingest-scoped, dedup, bounded)`

## Testing strategy

- Unit: schema validation, email resolution, bounds.
- Integration: full POST → 202 with counts; 401/403 auth; 413 oversize; duplicate handling; tenancy (rows carry key's org).
- Mock the celery broker (`send_task`) — assert task name + args; do not require a live worker.
- `cd services/backend-api && pytest tests/ -v`.

## Edge cases & errors

- No `messageId` → per spec: count `no_message_id` (default) — confirm vs hash fallback at build time.
- Bad/absent `timestamp` → use server `received_at`.
- Oversized `properties` → truncate + flag, still ingest.
- Auth: missing/invalid key → 401; valid key without `ingest` scope → 403.
- Cross-tenant: writes hard-scoped to `auth.organization_id`; never trust an `organization_id` in the body.
