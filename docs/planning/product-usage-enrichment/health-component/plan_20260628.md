- Unit: characterization (stability), component mapping (stub rollup), validator (sum=100), no-data→50.

- Migration round-trip: `alembic upgrade head && alembic downgrade -1 && alembic upgrade head`.

## Edge cases & errors

- `customer_usage` table/row missing → `_compute_usage_component` returns 50 (never raises).
- Org with no `org_ai_config` row → defaults incl. usage 0 (existing fallback path).
- Multi-tenancy: all reads scoped by `organization_id` (unchanged).
