## Phase 1 — model + migration
- **Files:** `services/backend-api/src/models/salesforce_integration.py` (new) + register in `models/__init__.py`; worker mirror in `services/worker-service/src/models/__init__.py`; `alembic/versions/<rev>_add_salesforce_integrations_table.py` (`down_revision=<aspect-1 rev>`).

- **Validation:** migration round-trip; model imports; parity test if one is added for salesforce.
- **Commit:** `feat(salesforce): salesforce_integrations model + migration`

## Phase 2 — shared helpers: one-CRM guard + purge
- **Files:** `services/backend-api/src/services/crm_integration_common.py` (new).
- **Steps:**
  1. `another_crm_active(db, org_id, exclude_provider) -> Optional[str]`: returns the name of any OTHER active CRM (`hubspot_integrations` / `salesforce_integrations` with `is_active=True`), else None.
  2. `purge_crm_enrichment(db, org_id, provider)`: delete `crm_enrichment` rows for `(org, provider)`, collect affected emails, call `update_customer_health(org_id, email, db)` for each (guarded import). 
- **Tests (RED):** guard detects the other provider; purge deletes only that provider's rows + triggers recompute (mock health).
- **Commit:** `feat(crm): shared one-CRM guard + provider purge helpers`

## Phase 3 — OAuth connect-url + status (GREEN)
- **Files:** `services/backend-api/src/api/routes/salesforce_integration.py` (new); register in `src/api/main.py`.
- **Steps:**
  1. Router prefix `/api/v1/integrations/salesforce`, all `require_admin_or_owner` + `require_feature("salesforce_integration")`.
  2. `GET /connect-url`: if `another_crm_active(...,exclude="salesforce")` → 409. Else build Salesforce authorize URL (`{login_base}/services/oauth2/authorize?response_type=code&client_id=...&redirect_uri=...&scope=refresh_token%20offline_access%20api&state=<signed>`); return `{auth_url}`. Sign `state` (reuse app secret / itsdangerous if present; else HMAC).
  3. `GET /status`: never returns tokens; returns connected/instance_url/sf_org_id/sync stats.
- **Tests:** connect-url shape + scopes + admin-only + 409-when-hubspot-active; status shape.
- **Commit:** `feat(salesforce): OAuth connect-url + status (ingest one-CRM guard)`

## Phase 4 — callback + test + disconnect (with purge)
- **Steps:**
  1. `GET /callback?code=&state=`: verify `state`; POST to `{login_base}/services/oauth2/token` (grant_type=authorization_code) → `access_token`, `refresh_token`, `instance_url`, `id`; derive `sf_org_id` from `id`; encrypt `refresh_token` via `encrypt_api_key` (R6 guard → 422); upsert row; redirect to `/settings/integrations/salesforce?connected=1` (or `?oauth_error=`). Re-check the one-CRM guard here too.
  2. `POST /test`: refresh token → probe `{instance_url}/services/data`; wrap in `{"success": bool}`, never 500.
  3. `DELETE /disconnect`: `is_active=False` + `purge_crm_enrichment(db, org_id, "salesforce")`.
  4. `POST /sync`: lazy celery `send_task("src.tasks.salesforce_sync.sync_salesforce_org", args=[integ.id])`.
- **Tests:** callback persists encrypted refresh_token (mock token exchange); test handles refresh failure as success:false; disconnect purges + recomputes (mock); missing encryption key → 422.
- **Commit:** `feat(salesforce): OAuth callback + test + disconnect(purge) + manual sync`

## Phase 5 — symmetric guard on HubSpot + plans + config
- **Files:** `services/backend-api/src/api/routes/hubspot_integration.py` (add `another_crm_active(...,exclude="hubspot")` → 409 in connect); `src/config/plans.py` (register `salesforce_integration` in 4 feature lists + `FEATURE_PLANS`, mirror `hubspot_integration`).
- **Tests:** HubSpot connect returns 409 when Salesforce active; feature id resolves unlocked under SELF_HOSTED.
- **Commit:** `feat(crm): symmetric one-CRM guard on hubspot connect + register salesforce_integration feature`

## Edge cases & errors
- `state` CSRF: sign + verify; reject mismatched.
- `invalid_grant` at callback → 422 with clear message.
- Missing `LLM_ENCRYPTION_KEY` → 422 (R6), never 500.
- Cross-tenant: writes hard-scoped to `auth`/current org; never trust body org.
- Mock ALL Salesforce HTTP in tests.
