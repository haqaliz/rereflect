# Implementation Plan — urgency-core (analysis-engine)

**Aspect:** `urgency-core` · **Feature:** `urgency-classifier-head` · **Date:** 2026-07-14
**Spec:** `docs/planning/urgency-classifier-head/urgency-core/spec.md`
**Service:** `services/analysis-engine`
**Sequence:** FIRST — foundation for worker-trainer. No dependencies.

## 0. Summary

Add a **binary urgency vocabulary** and a **dataset builder** to the already-generic
`corrections_classifier` package, so the urgency head trains/evaluates on the same primitives as
sentiment (fixed-vocab) — NOT category (dynamic-vocab). This is the smallest aspect: **3 edits + tests**,
zero changes to `trainer.py` / `predict.py` / `evaluate.py` / `metrics.py`.

Confirmed against source:
- `predict.py:105-110` already has the binary sigmoid path (`len(coef)==1` → `classes[1]` is positive).
- `rows_to_dataset(rows, allowed_labels=...)` already does fixed-vocab filtering (sentiment passes the
  tuple, category passes `None`). Urgency = pass `URGENCY_LABELS` (fixed 2-tuple).
- `train_classifier(dataset, classifier_type="urgency")` just tags the artifact — no code change.

## 1. Project Setup

- No new deps, no migration, no `.env`. Pure edits within `corrections_classifier/`.
- Python: analysis-engine venv (`services/analysis-engine`, `pytest`).

## 6. Edge Cases & Error Handling

- **Out-of-vocab `corrected_value`** (e.g. `"maybe"`, `"URGENT"` uppercase, `""`): dropped by
  `allowed_labels=URGENCY_LABELS` + `_normalize_label` strip. (Note: `_normalize_label` only strips, does
  not lowercase — capture-seam MUST write exactly `"urgent"`/`"not_urgent"` lowercase. Covered by a test.)
- **No resolvable text:** dropped (existing `_resolve_text` behavior).
- **Single-class dataset** (all `urgent` or all `not_urgent`): out of scope for core — `evaluate`'s
  `MIN_LABELS`/holdout guards handle degenerate sets; add a Phase-4 note but do not special-case here.
- **Empty dataset:** `rows_to_dataset([]) == []`; downstream `evaluate` handles insufficient data.
