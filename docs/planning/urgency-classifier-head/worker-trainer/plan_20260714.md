## 2. Implementation Phases (strict TDD)

### Phase 1 — Binary urgency incumbent predictor

- **Goal:** a deterministic `text -> "urgent" | "not_urgent"` predictor faithful to the production heuristic.
- **Files:** `classifier_training.py`; tests `services/worker-service/tests/` (mirror the existing
  `_build_incumbent_predict` / `_build_category_incumbent_predict` tests).
- **RED:** characterization tests:
  - urgent-keyword + very-negative text (compound < -0.5) → `"urgent"`.
  - urgent-keyword but mild/positive sentiment → `"not_urgent"` (the `and` requires very-negative).
  - very-negative but NO urgent keyword → `"not_urgent"`.
  - a table of cases cross-checked against the literal heuristic (keyword list + `< -0.5`).
- **GREEN:** add, mirroring `_build_incumbent_predict()` (`:70`, reuses cached
  `get_sentiment_analyzer("vader")`):
  ```python
  _URGENT_KEYWORDS = ('urgent','critical','broken','crash','bug','error','failing','down','cannot',"can't","won't","doesn't")

  def _build_urgency_incumbent_predict() -> Callable[[str], str]:
      """Binary incumbent = the production keyword+sentiment urgency heuristic
      (worker analysis.py:559-566 / backend feedback.py:107-112), wrapped as text->label.
      Reuses the cached VADER analyzer for the compound score; emits URGENCY_LABELS."""
      from src.tasks.analysis import get_sentiment_analyzer
      analyzer = get_sentiment_analyzer("vader")
      def _predict(text: str) -> str:
          text_lower = (text or "").lower()
          has_kw = any(k in text_lower for k in _URGENT_KEYWORDS)
          compound = analyzer.analyze(text)["compound"]
          return "urgent" if (has_kw and compound < -0.5) else "not_urgent"
      return _predict
  ```
  **Faithfulness note:** the ingest path's `strong_negative_keywords` neutral→-0.1 nudge (analysis.py:545-556)
  only lifts scores into the (-0.5, -0.1] band, which is never "very negative", so it can't change the
  urgent decision — using raw VADER compound here is faithful. Add a comment saying so.
- **Validation:** `pytest tests/ -k urgency_incumbent -v` (worker).
- **Commit:** `feat(urgency-trainer): binary urgency incumbent (keyword+sentiment heuristic wrapper)`

## 6. Edge Cases & Error Handling

- **R-3 class imbalance:** macro-F1 (not accuracy) is the promotion metric via `evaluate` — the collapse
  test guards it. Do NOT special-case; rely on the existing gate.
- **R-4 sparsity:** below `MIN_LABELS` → the head stays on the incumbent (skipped), eval-run recorded.
  Honest, not a failure.
- **Incumbent fidelity:** any drift from the literal keyword list / `-0.5` threshold makes the A/B bar a
  strawman — the characterization table is the guard. If the production heuristic ever changes, this
  incumbent must change with it (add a comment cross-referencing analysis.py:559-566).
- Empty/whitespace text → incumbent returns `"not_urgent"` (no keyword, guarded `(text or "")`).
