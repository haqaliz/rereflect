## 1. Locked decisions

1. **Column name/shape:** `org_ai_config.category_classifier_mode` — `String(20) NULL server_default='off'`, ORM `Column(String(20), nullable=True, server_default='off', default='off')`. Byte-identical shape to `classifier_mode`, just a new sibling column — not a rename, not a shared column.
2. **Validation set:** reuse `ai_settings.py`'s existing `VALID_CLASSIFIER_MODES = {"off","shadow","auto"}` for category too (same 3 values, same semantics: off/shadow/auto). One constant, two fields validate against it — do not create `VALID_CATEGORY_CLASSIFIER_MODES`, that would be a spurious duplicate of an identical set.
3. **Sklearn guard:** reuse `_classifier_deps_available()` unchanged — category head trains via the same TF-IDF + LogisticRegression scikit-learn path, so the same dependency gate applies to `category_classifier_mode in ('shadow','auto')`.
4. **Migration is additive-only:** one `op.add_column`, one `op.drop_column` in `downgrade()`. No changes to `org_classifier_models` / `org_classifier_eval_runs` (already `classifier_type`-generic per PRD §Technical Considerations) and no changes to `ai_corrections` (already flows `correction_type='category'`).
5. **`resolve_classifier` / `apply_classifier_override` / frontend are explicitly NOT touched by this aspect.** They currently read only `classifier_mode` regardless of `classifier_type` (confirmed at `classifier_resolver.py:78`) — that per-type branch is predict-seam's job (blocked on this aspect landing first). Wiring the new column into resolution before predict-seam ships would be scope creep and would have zero test coverage from *this* aspect's spec (whose acceptance criteria stop at GET/PATCH persistence).
6. **Worker-service does not run its own Alembic migrations** (shared Postgres DB; backend-api's `alembic upgrade head` is the only migration runner) — worker only needs the ORM column mirror, never a migration file of its own. Confirmed by the existing `classifier_mode` precedent (worker has the column, no worker-side migration exists).

    fd, path = tempfile.mkstemp(suffix=".db")

    engine._test_db_path = path  # type: ignore[attr-defined]
    return engine


def _dispose(engine):
    engine.dispose()
    path = getattr(engine, "_test_db_path", None)
    if path and os.path.exists(path):

        _migration.downgrade()
