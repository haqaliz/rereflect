## 1. Locked design decisions

| # | Decision | Rationale |
|---|---|---|
| D1 | `classifier_resolver.py` gets a module constant `MODE_COLUMN_BY_CLASSIFIER_TYPE = {"sentiment": "classifier_mode", "category": "category_classifier_mode"}`; `resolve_classifier` looks up the column name from `classifier_type`, then `getattr`s it off `config`. Unknown `classifier_type` → `None` (same contract as unrecognized mode). | Minimal, extensible (a future `urgency` type is a one-line dict addition), preserves the existing never-raises / missing-column-degrades-to-off posture. |
| D2 | `apply_classifier_override` branches on `classifier_type == "category"` in exactly 3 places: (a) which predict method to call on `LoadedClassifier` (new `predict_label_only`, no score), (b) what "incumbent" value to log, (c) which field (if any) to mutate in `auto`. The `else` branch is the **original, untouched sentiment code path** — same variable names, same log keys, same mutation. | Guarantees "sentiment override behavior byte-unchanged" without needing a snapshot/diff test — the sentiment branch is literally the pre-existing code, unindented into an `else`. |
| D3 | New `LoadedClassifier.predict_label_only(text) -> str` method: calls `analyzer.corrections_classifier.predict.predict()` directly and returns just the label, skipping `score_from_proba`. `predict()` (sentiment) is untouched. | Spec: "Bypass `score_from_proba` for category (no signed axis); do not write a sentiment score." Understanding.md Q3: "leave the existing category field's companion score untouched." |
| D4 | New module-level helper `_route_category_label(label) -> Optional[str]` returns `"pain_point_category"` / `"feature_request_category"` / `None` (ambiguous — neither or both) by checking membership in `PainPointCategorizer._BASE_CATEGORIES` / `FeatureRequestCategorizer._BASE_CATEGORIES` (`services/analysis-engine/src/analyzer/categorizer.py:21,211` — imported lazily, same pattern as the existing `LoadedClassifier.predict` lazy import). | Spec's "Unambiguous-routing rule" (critique #1) + "keep the routing table in one shared spot to avoid drift" — reusing `categorizer.py`'s own class-level dicts as the single source means there is **no second vocab list to maintain or let drift**; both service mirrors reach the same analysis-engine file. |
| D5 | Shadow-log `extra` dict gains one new key, `target_field` (`None` for sentiment, and for category: the routed field or `None` if ambiguous). No existing keys removed; sentiment's `incumbent_label`/`incumbent_score`/`challenger_label`/`challenger_score` keys and values are unchanged for `classifier_type="sentiment"`. | Confirmed no existing test asserts `extra` dict contents (`grep` for `incumbent_label`/`challenger_score` across all classifier tests → 0 matches) — only record **count** per logger name is asserted. Safe additive change; gives ops visibility into *why* an item wasn't routed. |
| D6 | Backend call-site (`routes/feedback.py`) adds a **shadow-only** (`allow_override=False`) category call, ordered **before** the existing sentiment call, right after the urgent-categorization block. Worker call-sites (`tasks/analysis.py`, both branches) add an **authoritative** (`allow_override=True`) category call immediately **after** each existing sentiment call, inside the same `if db is not None:` guard, reusing the already-imported `apply_classifier_override` name (one `from ... import` per call-site, two calls). | Matches spec's placement (`~:147` backend, beside `:427-429`/`:604-606` worker) and the existing ownership split (worker = sole `auto` writer for both types; backend inline path never mutates, even in `auto`). |
| D7 | The category override runs **unconditionally** per item (same as sentiment) — it does **not** require the base keyword/LLM categorizer to have set `pain_point_category`/`feature_request_category` first. If the challenger routes to a field, `auto` overwrites that field's current value (which may be `None`, e.g. on a neutral item the keyword path never touched). | Mirrors sentiment: `apply_classifier_override(..., "sentiment", ...)` runs regardless of which keyword-sentiment branch fired. Confirmed by acceptance criteria language "`pain_point_category` is overwritten" (not "only if already set"). **Flag this explicitly in code review / PR description** — it's a deliberate, spec-driven behavior, not an oversight. |

```python
    def predict_label_only(self, text: str) -> str:
        """Predict just the label for `text`, bypassing score_from_proba.

        Used for classifier_type="category": there is no signed axis
        (positive/negative poles) to reduce through score_from_proba, so
        this skips that step entirely and returns only the argmax label.
        Same lazy import as predict() — importing this module never
        requires analyzer.corrections_classifier.predict until a prediction
        is actually made.
        """
        from analyzer.corrections_classifier.predict import predict as _predict

        label, _proba = _predict(self.artifact, text)
        return label
```

### 3.2 Add `_route_category_label` (module-level function, after `load_active_classifier`, before `apply_classifier_override` — i.e. after line 184)

```python
def _route_category_label(label: str) -> Optional[str]:
    """Map a predicted category label to the FeedbackItem field it should
    override, by built-in-vocab membership (unambiguous-routing rule).

    Reuses analyzer/categorizer.py's built-in keyword-category dicts as the
    single shared vocab source (no second copy to let drift between the
    backend and worker mirrors — see predict-seam spec's open question).

    Returns:
        "pain_point_category" if `label` is in PainPointCategorizer's
            built-in vocab and NOT FeatureRequestCategorizer's.
        "feature_request_category" if the reverse.
        None if `label` is in NEITHER built-in vocab (e.g. a custom
            category the org's corrections introduced) or in BOTH
            (should not happen with today's disjoint base vocabs, but
            defended anyway) — the caller must not guess a field to
            write; shadow-log only.
    """
    from analyzer.categorizer import PainPointCategorizer, FeatureRequestCategorizer

    in_pain = label in PainPointCategorizer._BASE_CATEGORIES
    in_feature = label in FeatureRequestCategorizer._BASE_CATEGORIES

    if in_pain and not in_feature:
        return "pain_point_category"
    if in_feature and not in_pain:
        return "feature_request_category"
    return None
```

Note: `_BASE_CATEGORIES` is a class attribute (not instance `CATEGORIES`, which may include
`add_custom_categories()` merges elsewhere in the codebase) — deliberately using the class attribute so
this routing table reflects only the **built-in** vocab, per spec, regardless of what any particular
categorizer instance has been extended with.

```python
    try:
        from src.services.classifier_resolver import resolve_classifier

        org_id = getattr(feedback, "organization_id", None)
        if org_id is None or db is None:
            return

        resolved = resolve_classifier(org_id, classifier_type, db)
        if resolved is None:
            return  # off / unconfigured — no-op.

        loaded = load_active_classifier(org_id, classifier_type, db)
        if loaded is None:
            return  # No promoted model / corrupt artifact — incumbent retained.

        text = getattr(feedback, "text", None) or ""

        if classifier_type == "category":
            challenger_label = loaded.predict_label_only(text)
            challenger_score = None
            target_field = _route_category_label(challenger_label)
            incumbent_label = getattr(feedback, target_field, None) if target_field else None
            incumbent_score = None
        else:
            challenger_label, challenger_score = loaded.predict(text)
            target_field = None
            incumbent_label = getattr(feedback, "sentiment_label", None)
            incumbent_score = getattr(feedback, "sentiment_score", None)

        shadow_logger.info(
            "classifier shadow prediction",
            extra={
                "org_id": org_id,
                "feedback_id": getattr(feedback, "id", None),
                "classifier_type": classifier_type,
                "incumbent_label": incumbent_label,
                "incumbent_score": incumbent_score,
                "challenger_label": challenger_label,
                "challenger_score": challenger_score,
                "model_id": loaded.model_id,
                "fit_at": loaded.fit_at.isoformat() if loaded.fit_at else None,
                "target_field": target_field,
            },
        )

        if resolved.mode == "auto" and allow_override:
            if classifier_type == "category":
                if target_field is not None:
                    setattr(feedback, target_field, challenger_label)
                # else: ambiguous (neither/both built-in vocab) — shadow-

            else:
                feedback.sentiment_label = challenger_label
                feedback.sentiment_score = challenger_score
        # shadow, or auto+allow_override=False (backend inline ownership) —
        # log-only, no mutation.

    except Exception as exc:

            "apply_classifier_override: failed for classifier_type=%s: %s",

        )
```
