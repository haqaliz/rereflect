# Implementation Plan ‚Äî frontend (2026-07-06)

**Aspect:** `asana-integration/frontend` ¬∑ **Service:** `services/frontend-web`
**Depends on:** backend-connection + backend-create-task (API contracts). **Precedent:** `lib/api/jira.ts`, the Jira connect page, the create-issue wizard.

## 1. Setup checklist
- Service: `services/frontend-web` only. No new packages (reuse `apiClient`, shadcn/ui, Vitest).
- Asana specifics vs Jira: connect = **PAT only** (no site URL/email); wizard = **Workspace ‚Üí Project pickers** (no issue-type); brand `#F06A6A`.

## 4. Testing strategy
Vitest: `asana.test.ts` (client), `asanaIssueWizard.test.ts` (helpers). Command: `cd services/frontend-web && npm run test && npm run lint`.
Manual E2E (after backend running via `./start-all.sh`): Settings ‚Üí Integrations ‚Üí Asana ‚Üí paste PAT ‚Üí connected card; open a feedback ‚Üí Actions ‚Üí Create Issue ‚Üí Asana tile active ‚Üí workspace/project populate ‚Üí submit ‚Üí task in Asana; retry same feedback ‚Üí "Create anyway"; revoke token ‚Üí "Reconnect Asana".

## 5. Environment & secrets
`services/frontend-web/.env.local` `NEXT_PUBLIC_API_URL` points at the backend (already set). No new vars.

## 6. Edge cases & error handling
- PAT-only connect; no site/email fields.
- Large workspace/project lists ‚Äî flat `Select` (accepted slice-1 limit; team-scoped projects may be missing ‚Üí documented in landing/docs).
- 403 anywhere (connect/test/metadata/create) ‚Üí "Reconnect Asana", never a raw error.
- Duplicate ‚Üí 200 warning + "Create anyway".
- Colors via CSS variables / brand hex only where the Jira tile uses hex.
