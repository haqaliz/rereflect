## 2. Phases

### Phase 1 — Extract shared customer-filter helper (refactor, characterization-gated)
- **Files:** `src/api/routes/customers.py`.
- **Steps:**
  1. Characterization test: snapshot `list_customers` results across param combos (segment, risk_level,
     search, include_archived, sort). (Locks behavior.)
  2. Extract the filter-building block (`customers.py:288-319`) into
     `_apply_customer_filters(query, org, *, segment, risk_level, search, include_archived)` returning the
     filtered query; `list_customers` calls it. Sorting can stay in the route or move too (keep minimal).
  3. Run characterization test → GREEN (byte-identical).
- **Validation:** `pytest tests/ -v -k customers`.
- **Commit:** `refactor(segment-actions): extract _apply_customer_filters (behavior unchanged)`

### Phase 2 — Cohort schema + resolver
- **Files:** `src/schemas/cohort.py`, `src/services/cohort_service.py`, tests.
- **Steps:**
  1. RED tests: `Cohort` validator (exactly-one of emails/filter); `resolve_cohort` by emails
     (normalize/skip unknown/cross-org) and by filter (parity with `list_customers`).
  2. `Cohort {emails: list[str]|None, filter: CohortFilter|None}`, `CohortFilter {segment?, risk_level?,
     search?, include_archived?=False}`; `model_validator` → exactly one set else `ValueError` (→422).
  3. `resolve_cohort(db, org, cohort)`: emails → normalized, `organization_id==org.id`,
     `customer_email.in_(...)`, return `(rows, skipped_count)`; filter → `_apply_customer_filters(...)` on an
     org-scoped base query. Provide `count_cohort(...)` cheap count helper.
- **Validation:** `pytest tests/ -v -k cohort`.
- **Commit:** `feat(segment-actions): Cohort schema + resolve_cohort service`

### Phase 3 — CSV export endpoint
- **Files:** `src/api/routes/customers.py` (register before `/{email}`), tests.
- **Steps:**
  1. RED test: `GET /customers/export?segment=at_risk` → 200, `text/csv`, attachment header, header row +
     rows for at-risk only, org-scoped, no sentiment_trend column.
  2. Implement `export_customers(...)` with same query params as list; keep
     `require_feature("customer_health_scores")`. Build a generator that paginates via
     `_apply_customer_filters` (batch size ~500), yields CSV rows (stdlib `csv.writer` over a `StringIO`
     per chunk). Columns per spec (omit sentiment_trend). `StreamingResponse(gen, media_type="text/csv",
     headers={"Content-Disposition": f'attachment; filename="customers-{seg or "all"}.csv"'})`.
  3. Route order: ensure `/export` registered before `/{email}` (place above, or dedicated router included
     first). Add to `docs/API.md` + docstring.
- **Validation:** `pytest tests/ -v -k export`; assert no per-row sentiment-trend call (spy/patch).
- **Commit:** `feat(segment-actions): GET /customers/export streaming CSV`

### Phase 4 — Bulk tag + assign-owner endpoints
- **Files:** `src/schemas/cohort.py` (or `customer_bulk.py` schemas) for `BulkActionSummary`,
  `src/api/routes/customers.py`, tests.
- **Steps:**
  1. RED tests: `/bulk/tags` add/remove (union/difference, caps, org scope, skip cross-org, summary);
     `/bulk/assign-owner` set/clear, non-member user_id → 422, org scope, summary; both require admin/owner
     (member → 403).
  2. `BulkActionSummary {matched:int, updated:int, skipped:int, errors:list[str]}`.
  3. `POST /customers/bulk/tags` (`dependencies=[Depends(require_admin_or_owner)]`): resolve cohort → per
     row apply add/remove with validation (trim/dedupe/≤50 chars/≤20 total) → commit → summary. Registered
     before `/{email}`.
  4. `POST /customers/bulk/assign-owner` (`require_admin_or_owner`): validate `user_id` is active org member
     (query org membership) or null; resolve cohort → set `cs_owner_user_id` → commit → summary.
  5. Docs: both endpoints in `docs/API.md` + docstrings.
- **Validation:** `pytest tests/ -v -k "bulk_tag or assign_owner"`.
- **Commit:** `feat(segment-actions): POST /customers/bulk/tags + /bulk/assign-owner`

## 6. Edge cases
- Both/neither of emails|filter → 422 (validator).
- Cross-org / unknown emails → skip, counted in `skipped`.
- Tag caps: >20 after union → trim to 20 or reject? **Reject the over-cap customer into `errors`** (don't
  silently drop) — confirm in test.
- assign-owner to a user in another org → 422.
- Streaming: keep DB session valid for the generator lifetime; paginate (no full-table materialize).
- Route ordering: static paths MUST precede `/{email}` (else `export`/`bulk` captured as an email).
