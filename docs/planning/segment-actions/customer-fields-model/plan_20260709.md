### Phase 2 — Model
- **Files:** `src/models/customer_health.py`.
- **Steps:**
  1. Add `tags = Column(JSON, nullable=True, default=list)` (import `JSON` from sqlalchemy; `default=list`
     callable — never a shared `[]`).
  2. Add `cs_owner_user_id = Column(Integer, ForeignKey("users.id", ondelete="SET NULL"), nullable=True)`.
  3. Add relationship `cs_owner = relationship("User", foreign_keys=[cs_owner_user_id], lazy="joined")` (or
     `selectin`); confirm `User` needs no back_populates and no cascade side-effects.
  4. Add index in `__table_args__` if the model declares them there (match migration name).
- **Validation:** `pytest tests/ -v -k customer_health` (model imports, defaults).
- **Commit:** `feat(segment-actions): CustomerHealth.tags + cs_owner_user_id model`

### Phase 3 — Serializers (list + profile) with batched owner load
- **Files:** `src/api/routes/customers.py` (`CustomerListItem` ~L40-52, `CustomerProfileResponse` ~L85-121,
  `list_customers` L252-364, profile route L367+).
- **Steps:**
  1. Write a characterization test capturing the current `GET /customers/` + `GET /customers/{email}` JSON
     shape (before change) → keep as regression guard. (RED for new fields.)
  2. Add `tags: list[str] = []` and `cs_owner: Optional[CustomerOwnerRef]` to `CustomerListItem` +
     profile response, where `CustomerOwnerRef = {id:int, email:str}` (the `User` model has NO name field —
     only `email` — so do not add `name`).
  3. In `list_customers`, after fetching the page rows, **batch-load** owner users: collect distinct
     non-null `cs_owner_user_id`, one `User` query `id.in_(...)`, build `{id: ref}` map (mirror the batched
     `CustomerUsage` fetch at L324-336). Populate per row. `tags` comes straight off the row (`row.tags or []`).
  4. Profile route: populate `tags` + `cs_owner` (single-row, direct).
- **Validation:** `pytest tests/ -v -k customers`; assert **no extra per-row owner query** (query-count
  assertion or an eager-load check); existing fields unchanged (characterization test green).
- **Commit:** `feat(segment-actions): expose tags + cs_owner on customers list/profile`
