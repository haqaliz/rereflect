## Scope recap (what this aspect ships)

Persist per-org classifier artifacts + eval history + the per-org mode toggle, mirroring the proven
`churn_calibration_models` / `ChurnBacktestRun` conventions, in **both** backend ORM and the worker
mirror, plus **one** Alembic migration. No training/predict/schedule/API/UI (aspects B–E).

Concrete deliverables:

1. Backend `OrgClassifierModel` + `OrgClassifierEvalRun` (new file `src/models/org_classifier.py`), exported from `src/models/__init__.py`.
2. Backend `OrgAIConfig.classifier_mode` `VARCHAR(20)` NULL default `'off'`.
3. One Alembic migration: create both tables + add `classifier_mode`. `down_revision='6ad1dc4335f1'`.
4. Worker mirror (`services/worker-service/src/models/__init__.py`): add `OrgClassifierModel`, `OrgClassifierEvalRun`, the **currently-missing `AICorrection`**, and `OrgAIConfig.classifier_mode`.
5. Cross-service column-parity characterization tests.

### Locked cross-cutting constraints (obeyed by this plan)

- `model_json` is **`JSON`, never pickle**.
- **CPU-only, no heavy imports** — this aspect touches only SQLAlchemy/Alembic; no `sklearn`/`numpy`/`torch` anywhere.
- Migration `down_revision = '6ad1dc4335f1'` (current single head).
- New models + `classifier_mode` mirrored into **both** services; `AICorrection` **added** to the worker mirror.
- Partial-unique "one active per `(organization_id, classifier_type)`" copies the churn model's dialect-aware form **exactly** (`sqlite_where=text("is_active = 1")`, `postgresql_where=text("is_active = TRUE")`).

## Edge Cases & Error Handling

1. **`classifier_type` sizing:** `String(30)` covers `sentiment`/`category`/`pain_point`/`feature_request` (v2). Fixed now to avoid a later migration.
2. **`original_value`/`corrected_value` type drift:** the `ai_corrections` **migration** (`s8t9u0v1w2x3`) declared these `String(500)`, but the backend **model** (`ai_correction.py`) uses `Text`. The parity test compares column **names**, not types, so mirroring the backend model's `Text` is correct and passes. (Flagged for awareness; not fixed in this aspect.)
3. **NULL `organization_id`:** allowed on both new tables (global/base model + global eval run) — explicitly tested.
4. **`classifier_mode` on un-migrated rows:** a newer app reading a pre-migration `org_ai_config` row: `getattr(config, 'classifier_mode', 'off')` returns `'off'`; downstream resolver (aspect D) must use that defensive getattr — tested here so the contract is locked at the data layer.
5. **Partial-unique on the write path:** the worker is the sole writer of `org_classifier_models` (trainer, aspect C); the DB-level partial-unique guard is the backstop for the insert-new-active + flip-old-inactive transaction (PRD "trainer/promotion atomicity"). Hence the worker mirror **keeps** the index (unlike the read-only `ChurnCalibrationModel` mirror which omits it).
6. **`is_active` server_default:** `sa.false()` in the migration (SQLite renders `0`, PG renders `FALSE`) — matches the churn migration; the partial index predicate `is_active = 1`/`= TRUE` aligns with that default.
7. **Downgrade FK order:** drop `org_classifier_eval_runs` (references models) **before** `org_classifier_models`; `classifier_mode` dropped first via plain `op.drop_column` (same as the `6ad1dc4335f1` downgrade, which the repo's SQLite/PG already support).
8. **Second Alembic head:** if `alembic heads` shows >1 after this migration, a concurrent branch split the tree — do **not** silently proceed; create a merge migration (explicitly out of scope unless it occurs).
