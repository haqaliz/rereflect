## Contract from data-layer (A) — what we read (verified against `data-layer/spec.md`)

- `OrgAIConfig.classifier_mode` VARCHAR(20) NULL `server_default='off'` — values `off|shadow|auto`.
  Read `getattr(config, 'classifier_mode', 'off')`-defensive (mirror `sentiment_provider`).
- `OrgClassifierModel`: `id`, `organization_id` (nullable=global), `classifier_type` (`'sentiment'`),
  `label_count` INT, `macro_f1`/`precision`/`recall`/`accuracy` Numeric(5,4) nullable, `fit_at`,
  `is_active` Bool; partial-unique one active per `(organization_id, classifier_type)`.
- `OrgClassifierEvalRun`: `id`, `organization_id`, `classifier_model_id` (nullable FK),
  `classifier_type`, `incumbent_macro_f1`/`challenger_macro_f1`/`macro_f1_delta` Numeric(5,4) nullable,
  `decision` (`promoted|retained|skipped`), `n` INT, `duration_ms`, `notes`, `created_at`.

## Decisions (locked for this plan)

- **Card placement:** mode toggle → **General tab** (`AISettingsGeneral`, new card beside Sentiment
  Engine); accuracy card → **co-located with `SentimentAccuracyCard` in the existing "AI Accuracy" tab**
  (`value="accuracy"`), rendered directly below it. (Spec OQ recommendation confirmed — no new tab.)
- **Route:** `GET /api/v1/settings/ai/classifier/accuracy`, new file
  `src/api/routes/classifier_accuracy.py`, `router = APIRouter(prefix="/api/v1/settings/ai",
  tags=["classifier-accuracy"])`, org-scoped via `get_current_org`. No `require_feature` gate (OSS
  disclosure surface, mirrors `sentiment_accuracy.py`).
- **Schemas** (`src/schemas/classifier_accuracy.py`): `ClassifierAccuracyResponse` +
  `ClassifierEvalRunSummary`.
- **Readiness count semantics:** the card's "count vs threshold" uses the **active model's
  `label_count`** (0 when no active model) — NOT live `corrections_by_type["sentiment"]` (that is
  AIReadinessCard's job; reading it here would couple to M5.0). `is_ready = count >= MIN_CLASSIFIER_LABELS`.
- **Threshold:** `MIN_CLASSIFIER_LABELS = 20`, defined in `classifier_accuracy.py` and echoed in the
  response so the frontend renders `N/20` without hardcoding. Reconcile to a single source with
  training-and-eval-core (C) when C lands.
- **`classifier_mode` validation:** `VALID_CLASSIFIER_MODES = {"off", "shadow", "auto"}` defined locally
  in `ai_settings.py` (resolver in aspect D is NOT a dependency). Light deps guard
  `_classifier_deps_available()` = `importlib.util.find_spec("sklearn") is not None`; `shadow`/`auto`
  require it (422 otherwise, honest message pointing to SELF_HOSTING). `off` always allowed.
- **Rollback:** included as optional Phase 3 (cheap — single-table active flip, mirrors churn
  versioning). It is the aspect's only write.

## Phase 3 — Rollback endpoint (backend, optional should-have — included)

- **RED:** ADD to `tests/test_classifier_accuracy_route.py` (or new
  `tests/test_classifier_rollback.py`):
  - Seed active model v2 + inactive prior v1 → POST `/classifier/rollback` flips v1 active, v2
    inactive; response reflects v1; exactly one active row remains (partial-unique respected).
  - No prior version (only one model) → deactivates the active model (disable), `has_model=false`
    afterward; idempotent-safe.
  - No model at all → 404 (nothing to roll back).
  - Requires admin/owner (member → 403); cross-org: cannot roll back another org's model (404).
- **GREEN:** ADD `POST /api/v1/settings/ai/classifier/rollback` to `classifier_accuracy.py`
  (`dependencies=[Depends(require_admin_or_owner)]`, `classifier_type` body/query default `sentiment`):
  deactivate current active, activate most-recent inactive prior (by `fit_at desc`) in **one
  transaction ordered deactivate→activate→commit** so the partial-unique index never sees 2 active.
  Return `ClassifierAccuracyResponse` (fresh state) so the card can refresh from the response.
- **Validate:** `pytest tests/test_classifier_accuracy_route.py -v` (or the rollback file).
- **Commit:** `feat(m5.2-settings): POST classifier/rollback (reactivate prior / disable, atomic flip)`

## Edge cases

- Un-migrated `OrgAIConfig` row → `getattr(config, 'classifier_mode', 'off')` returns `"off"`; card GET
  never 500s (null-state), mirroring `sentiment/status` + churn null-state.
- `shadow`/`auto` selected but sklearn absent → 422 with an honest install message; toggle surfaces
  `err.response.data.detail`.
- Numeric(5,4) DB values → coerce to `float` (or `None`) in the route (mirror churn_accuracy's
  `float(...) if x is not None else None`).
- Delta null / decision `skipped` ("held-out too small") → render honestly; never hide loss numbers.
- Theme: badges legible in light AND dark (CSS vars only — no hex/oklch literals).

## Agent notes / ambiguities to surface

1. **Readiness count source** — this card shows the *last-fit* `label_count`, so a new org with e.g. 15
   sentiment corrections but no fit yet shows `0/20`, not `15/20`. If the PM wants the pre-fit live
   count, wire `corrections_by_type["sentiment"]` from `ai_readiness` — deferred to avoid M5.0 coupling.
2. **`MIN_CLASSIFIER_LABELS=20`** is duplicated here; make training-and-eval-core (C) the single source
   and import once C lands.
3. **Depends on data-layer (A)** being merged: the models/column/partial-index must exist for tests to
   seed rows. If A is not yet merged, this aspect's tests are RED-blocked on import.
4. Do NOT add training/predict/schedule logic (aspects B–E) or a category head (v2) — read-only surface
   plus the one rollback flip.
