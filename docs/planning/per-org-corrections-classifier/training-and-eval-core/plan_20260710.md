## 0. Scope, home, and resolved ambiguities

### What this aspect delivers
A CPU-only, offline, per-org **sentiment** classifier core:

1. `build_sentiment_dataset(org_id, db) -> list[(text, label)]` — the one DB-touching **thin seam**
   (split into a pure transform + a lazy-SQL fetch).
2. `train_classifier(dataset) -> artifact_dict` — TF-IDF + LogisticRegression → **JSON** (never pickle).
3. `predict(artifact, text) -> (label, proba_dict)` — pure, reconstructs the linear model from JSON.
4. `score_from_proba(proba) -> float` — `clamp(P(positive) − P(negative), −1, 1)`.
5. `evaluate(dataset, incumbent_predict, challenger_artifact, *, knobs) -> EvalResult` — shadow-A/B
   with the promote/retain/skip decision. Disclosure only, never fails a build.

**Out of scope (other aspects):** model-row read/write + `is_active` flip + scheduling (aspect C,
worker-trainer); the `classifier_mode` flag + call-site injection + resolver + cache key (aspect D);
the accuracy API endpoint + Settings UI (aspect E). This aspect ships pure functions those aspects
**call**.

### Home (locked): `analysis-engine/src/analyzer/corrections_classifier/`

Analysis-engine test convention (confirmed): `from src.analyzer...`, **no conftest**, run with

### Resolved ambiguities (decisions, with rationale)

1. **"Reuse `eval_sentiment.py::compute_multiclass_metrics`" = port the verbatim math, do not import
   across services.** `compute_multiclass_metrics` / `confusion_to_binary_counts` /
   `_safe_precision_recall_f1_accuracy` live in `backend-api/scripts/eval_sentiment.py`. Analysis-engine
   cannot (and shouldn't) import a backend-api script. The codebase's **own stated precedent** is to
   copy this metrics math per module rather than share it (`eval_sentiment.py:44-50` documents copying
   it from `admin_backtest.py`, and `backtest_churn.py` keeps its own copy). We follow that precedent:
   `corrections_classifier/metrics.py` is a **verbatim port** of the three functions with a docstring
   citing `eval_sentiment.py` as the source of truth and the confusion shape
   `confusion[true][pred] = count`. "Reuse" therefore means identical math + identical confusion-matrix
   shape + identical macro-F1 definition — provably so via a cross-check test that pins outputs to the
   same numbers `run_provider` would produce.

2. **The pure core needs *no* heavy imports except in the trainer.** `predict`, `score_from_proba`,
   `metrics`, `evaluate`, and the dataset transform are **pure stdlib** (Python `math`, `re`, `random`).
   Only `train_classifier` imports scikit-learn (+ numpy) — **lazily, inside the function**, mirroring
   `worker-service/src/services/calibration_refit.py` (`_fit_isotonic` imports `numpy`/`sklearn` inside
   the fn; see its module docstring on Py3.14 CI venvs lacking wheels). This means shadow/`auto`
   predict at analyze-time (aspect D, hot path) never imports numpy/sklearn and the whole package is
   importable in a wheels-less venv. The importability guarantee is asserted with a subprocess test
   mirroring `tests/sentiment_providers/test_lazy_import.py` (stub `sys.modules['sklearn']=None`).

3. **`predict` is pure-Python (no numpy at predict time).** It re-implements the TF-IDF transform +
   logistic decision from the JSON artifact. To reproduce sklearn's own predictions exactly (an
   acceptance criterion), the trainer **pins deterministic, default-ish vectorizer params** and stores
   everything predict needs: `vocabulary {term:index}`, `idf[]`, `token_pattern`, `lowercase`,
   `ngram_range`, `norm`, `sublinear_tf`, `smooth_idf`, plus `logreg.coef`, `logreg.intercept`,
   `classes`, `multi_class`. LogisticRegression is pinned to
   `multi_class="multinomial", solver="lbfgs", random_state=0` for a deterministic softmax that pure
   Python can reproduce; the binary-`classes==2` shape (`coef.shape == (1, n)`) is handled via sigmoid.

4. **`build_sentiment_dataset` is split** into `rows_to_dataset(rows)` (pure, fully unit-tested with
   plain dicts) + `fetch_sentiment_correction_rows(org_id, db)` (lazy-SQL seam). The fetch uses a single
   model-agnostic `sqlalchemy.text()` query (lazy import) so analysis-engine never hard-depends on the
   backend/worker ORM model classes. This is the faithful mirror of the churn split (DB collection is a
   thin driver; the math is pure).

5. **Locked knobs** (from PRD R2 / spec): `MIN_LABELS = 20`, `HOLDOUT_FRAC = 0.2`, `MIN_HOLDOUT = 8`,
   `MARGIN = 0.02`. Labels fixed `("negative", "neutral", "positive")` (sorted = sklearn `classes_`
   order). Incumbent predictor is an **injected callable** `incumbent_predict(text) -> label`
   (aspect D/worker wires the live `SentimentAnalyzer`); `evaluate` stays agnostic.

def train_classifier(dataset, *, random_state=RANDOM_STATE):
    from sklearn.feature_extraction.text import TfidfVectorizer   # lazy
    from sklearn.linear_model import LogisticRegression           # lazy
    texts = [t for t, _ in dataset]; labels = [l for _, l in dataset]
    vec = TfidfVectorizer(lowercase=True, ngram_range=(1, 1), norm="l2",
                          sublinear_tf=False, smooth_idf=True)
    X = vec.fit_transform(texts)
    clf = LogisticRegression(multi_class="multinomial", solver="lbfgs",

    clf.fit(X, labels)

**GREEN — `predict.py` (pure):**
- `_tokenize(text, token_pattern, lowercase)`: `re.findall(token_pattern, text.lower() if lowercase else text)`.
- `_tfidf_vector(tokens, vocab, idf, sublinear_tf, norm)`: count in-vocab tokens; `tf = 1+log(c)` if
  sublinear else raw `c`; `val = tf * idf[vocab[term]]`; if `norm=="l2"` divide by `sqrt(Σ val²)`
  (guard zero). Returns `{index: val}` sparse dict.
- `_decision(vec, coef, intercept)`: `d[c] = intercept[c] + Σ_j coef[c][j]*vec[j]` over sparse indices.
- multinomial: `proba = softmax(d)`; binary (`len(coef)==1`): `p1 = sigmoid(d[0])`,
  `proba = {classes[1]: p1, classes[0]: 1-p1}`.
- `label = max(proba, key=proba.get)`.
- `score_from_proba(proba)`: `max(-1.0, min(1.0, proba.get("positive", 0.0) - proba.get("negative", 0.0)))`.

**REFACTOR:** extract `_softmax`/`_sigmoid`; add a numerically-stable softmax (subtract max). Docstring
the "reproduces sklearn exactly because params are pinned" contract.

**Sub-step 4b — evaluate (RED → GREEN):**

**`EvalResult` dataclass:**
```python
@dataclass(frozen=True)
class EvalResult:
    decision: str                 # "promoted" | "retained" | "skipped"
    n: int                        # evaluated (held-out or k-fold) label count
    incumbent_macro_f1: Optional[float]
    challenger_macro_f1: Optional[float]
    macro_f1_delta: Optional[float]
    notes: str
```

**Signature:**
```python
def evaluate(dataset, incumbent_predict, challenger_artifact, *,
             min_labels=MIN_LABELS, holdout_frac=HOLDOUT_FRAC,
             min_holdout=MIN_HOLDOUT, margin=MARGIN, random_state=RANDOM_STATE) -> EvalResult
```
`incumbent_predict: Callable[[str], str]` (injected — aspect D wraps live `SentimentAnalyzer`;
`evaluate` never constructs it). Challenger label = `predict(challenger_artifact, text)[0]`.

**Decision logic (locked):**
1. `len(dataset) < min_labels` → `EvalResult("skipped", n=len(dataset), None, None, None, "below min_labels")`.
2. Stratified split (pure, `random.Random(random_state)`), holdout size `= round(n*holdout_frac)`.
   - If `holdout_size < min_holdout` → **k-fold mode**: stratified k-fold (k = `ceil(min_holdout/…)`,
     ≥3), aggregate one confusion matrix per model across all folds so every row is evaluated once;
     effective `n = len(dataset)`.
   - Else single stratified holdout; `n = holdout_size`.
3. **Small-sample guard:** the evaluated set must contain **all 3 classes**. If not (or, in single-holdout
   mode, `n < min_holdout`) → still compute+report the delta, but force
   `decision="retained"`, `notes="held-out too small"`.
4. Build two confusions (`_empty_confusion` shape from `eval_sentiment`), score incumbent + challenger
   over evaluated rows, `compute_multiclass_metrics` each → `incumbent_macro_f1`, `challenger_macro_f1`,
   `macro_f1_delta = challenger − incumbent`.
5. If guard passed and `macro_f1_delta >= margin` → `"promoted"`, else `"retained"`.

**RED — `test_evaluate.py`** (challenger artifacts built via a tiny `train_classifier`, guarded by
`importorskip("sklearn")`; incumbent injected as a plain lambda/stub — no real analyzer needed):
- `test_below_min_labels_returns_skipped` (19 rows → `decision=="skipped"`, deltas None).
- `test_tiny_holdout_all_classes_forces_retained_with_note` — n just ≥ min_labels but held-out < 8 in a
  path that yields a too-small evaluated set → `decision=="retained"`, `notes=="held-out too small"`,
  delta still populated.
- `test_missing_class_in_holdout_forces_retained` (evaluated set missing a class) → retained + note.
- `test_clearly_better_challenger_is_promoted` — incumbent stub deliberately wrong on held-out,
  challenger trained to be right → positive delta ≥ margin → `"promoted"`.
- `test_worse_challenger_is_retained` — incumbent stub = gold labels (perfect), challenger noisy →
  delta < margin → `"retained"`.
- `test_delta_equals_margin_promotes` (delta == 0.02 → promoted; boundary is `>=`).
- `test_incumbent_predict_is_injected` — pass a spy callable, assert it's called once per evaluated row.
- `test_evaluate_never_raises_on_degenerate` — single-class dataset above min_labels → returns a result,
  no exception (disclosure ethos).
- `test_deterministic_split` — same inputs+seed → identical `EvalResult`.

**GREEN — `evaluate.py`:** implement `_stratified_split`, `_stratified_kfold_confusions`,

## 5. Edge Cases (must be covered)

1. **Text join:** `feedback_text` blank/None → fall back to joined `FeedbackItem.text`; both blank → drop.
2. **Cross-org join leak:** `entity_id` pointing at another org's feedback item must NOT resolve text.
3. **Out-of-vocab label** (`"mixed"`, `""`, `"POSITIVE "` uppercase/whitespace) → normalized/dropped
   per rule (decide: strip+lowercase then vocab-check; document that corrected_value is stored lowercase
   already — normalize defensively).
4. **Below `MIN_LABELS`** (dataset or held-out) → `skipped` / guarded `retained`.
5. **Single-class or two-class dataset** → trainer still fits (binary `coef` shape); predict handles
   sigmoid path; evaluate guard forces `retained` if evaluated set lacks all 3 classes.
6. **Empty dataset** → builder returns `[]`; `evaluate([])` → `skipped`; `train_classifier([])` is never
   called (guarded upstream by aspect C, but note it here).
7. **All-out-of-vocab predict text** → decision driven by intercepts; no crash, valid proba.
8. **Zero-norm tf-idf vector** (text with no in-vocab tokens after norm) → guard divide-by-zero.
9. **Delta exactly at margin** → promoted (`>=`).
10. **Degenerate confusion** (all-zero / single-class) → metrics return 0.0, never raise.
11. **No pickle anywhere** → recursive JSON-type assertion + `\x80` byte scan.
12. **Missing class key in `score_from_proba`** → treated as 0.0; result clamped to `[-1, 1]`.
