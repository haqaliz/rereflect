## Phase 2 — RED: write failing tests for tags + is_urgent

- **Goal:** failing tests describing the new behavior.
- **Files:** `tests/test_public_api_write.py` — add a `TestPublicWriteTags` and `TestPublicWriteUrgent` class using existing helpers `_make_api_key(db, org_id, scopes=..., name=...)` and `_feedback_in_org(db, org_id, ...)`.
- **Cases (each hits `client.patch("/api/public/v1/feedback/{id}", json=..., headers=write_key)`):**
  1. `tags: ["Bug", " bug ", "perf"]` → 200, and **re-fetch via GET** (or a fresh `db.query`) shows `["Bug", "bug", "perf"]`? No — trim happens then dedupe is case-sensitive per spec: `" bug "`→`"bug"`, distinct from `"Bug"`. Assert persisted `["Bug","bug","perf"]`. (Proves JSON persistence + trim + order.)
  2. `tags: []` → 200, persisted `[]` (clears). Seed the item with existing tags first.
  3. PATCH without `tags` key (only `is_urgent: true`) → existing tags unchanged.
  4. `tags` with a duplicate (`["a","a"]`) → persisted `["a"]`.
  5. `tags` length 21 → 422. Element length 51 → 422. Non-string element (`["a", 3]`) → 422. Empty-string element (`["a",""]`) → 422 (after trim, empty rejected).
  6. `is_urgent: true` → 200, persisted True; `is_urgent: false` → persisted False; omitted → unchanged.
  7. Unknown field `{"tag": ["x"]}` → 422 (`extra="forbid"`).
  8. PATCH `{"tags": ["x"]}` only → 200 (guard widened), response `tags == ["x"]`.
  9. Combined `{"workflow_status":"resolved","tags":["x"],"is_urgent":true}` → 200, all three applied.
  10. Scope/tenant: read-key → 403, ingest-key → 403, cross-org id → 404 (reuse slice-1 patterns).
- **Validation:** `pytest tests/test_public_api_write.py -v` → new tests FAIL, old tests still PASS.

## Phase 3 — GREEN: implement

- **File:** `services/backend-api/src/api/routes/public_api.py`
- **Steps:**
  1. On `PublicFeedbackUpdate` add:
     - `model_config = {"extra": "forbid"}`
     - `tags: Optional[list[str]] = None`
     - `is_urgent: Optional[bool] = None`
     - A `@field_validator("tags")` (mode="after"): if `None` return `None`; else require every element `isinstance str`; `.strip()` each; drop empties; dedupe preserving first-seen order; enforce `len ≤ 20` and each `len ≤ 50` → raise `ValueError` (→ 422). (Import `field_validator` from pydantic.)
  2. In `public_update_feedback`:
     - Replace the guard `if data.workflow_status is None and data.correction is None:` with a check on `data.model_fields_set` — reject only if **none** of `{workflow_status, correction, tags, is_urgent}` were provided (`if not (data.model_fields_set & {"workflow_status","correction","tags","is_urgent"}): 400`).
     - After the workflow-status block and before/after the correction block, add a **field-mutation block**:
       ```python
       fields_touched = False
       if "is_urgent" in data.model_fields_set:
           fb.is_urgent = bool(data.is_urgent)
           fields_touched = True
       if "tags" in data.model_fields_set:
           fb.tags = list(data.tags or [])   # rebind new list → JSON dirty-tracked
           fields_touched = True
       if fields_touched:
           db.commit()
           # best-effort side effects (never fail the write)
           try:
               from src.services.cache_service import cache_invalidate
               cache_invalidate(f"dashboard:{auth.organization_id}:*")
               cache_invalidate(f"analytics:{auth.organization_id}:*")
           except Exception:
               logger.warning("cache_invalidate failed on public PATCH (fields)", exc_info=True)
           try:
               from src.services.event_emitter import emit_event
               await emit_event(org_id=auth.organization_id, event_type="feedback:updated", data={"id": fb.id})
           except Exception:
               logger.warning("emit_event failed on public PATCH (fields)", exc_info=True)
       ```
       (`model_fields_set` distinguishes `tags: []` = clear from omitted = untouched. `list(data.tags or [])` rebinds so SQLAlchemy sees the change; `flag_modified(fb, "tags")` is the alternative if a shared reference ever sneaks in.)
  3. Add `tags: Optional[list[str]] = None` to `PublicFeedbackItem` (response echo).
  4. Update the PATCH `description=` to mention tags/is_urgent (or leave for `docs-openapi`).
- **Validation:** `pytest tests/test_public_api_write.py -v` → all green (new + old).
