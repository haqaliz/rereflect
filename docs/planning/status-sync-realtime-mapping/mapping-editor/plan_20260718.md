## 6. Edge Cases & Error Handling
- Unset mapping → `null` from API → editor shows default targets unselected or
  the server-default hint; Save of a partial mapping is allowed (backend merges
  over default).
- Non-admin: editor read-only/hidden (mutations already 403 server-side).
- Invalid target (shouldn't happen via Select) → backend 422 (`_validate_status_mapping`).
- Zendesk already returns `status_mapping`; don't double-handle.
