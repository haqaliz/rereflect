### Phase 3 — Receiver (RED → GREEN → REFACTOR)
- **Goal:** real-time inbound events reconcile via shared core + race-safe apply.
- **Files:** new `services/backend-api/src/api/routes/jira_webhook.py`;
  register in `services/backend-api/src/api/main.py`;
  new backend-api reconcile port `services/backend-api/src/services/jira_status_reconcile.py`
  (mirror `zendesk_status_reconcile.py`, reuse the guarded UPDATE from the
  race-guard aspect); `tests/test_jira_webhook.py`.
- **Steps:**
  1. RED (`test_jira_webhook.py`, mirror `test_zendesk_webhook.py` +
     `test_linear_webhook.py`):
     - Missing/invalid `X-Hub-Signature` → 401; missing stored secret → 401
       (fail-closed — NOT Linear's fail-open).
     - Valid signature over raw bytes, org resolved by secret-match → processed.
     - `webhookEvent != "jira:issue_updated"` or no status changelog → 200 no-op,
       no writes.
     - Linked issue status change → `workflow_status` updated per merged mapping;
       exactly one `FeedbackWorkflowEvent` with
       `metadata={"source":"jira",...}`.
     - Stale/duplicate delivery (already applied) → zero events (race guard).
  2. GREEN: implement `POST /api/v1/webhooks/jira/inbound`:
     - Read raw body; `sig = X-Hub-Signature` (`sha256=<hex>`); iterate active
       `JiraIntegration` with a secret, `hmac.compare_digest`; 401 if none.
     - Parse JSON; extract issue key/id + new status name + `statusCategory`;
       discriminate to the status-change branch else 200.
     - `jira_status_reconcile.reconcile_issue(db, integ, issue_id, status_name,
       category)`: look up `FeedbackJiraIssue` links (org-scoped), compute
       most-advanced category, `resolve_target_status(cat, integ.status_mapping)`
       (shared `status_sync_core`), race-safe apply, upsert link last-observed;
       `db.commit()`.
     - Return `{"status":"ok","reconcile":result}`.
  3. REFACTOR: factor the reconcile so it mirrors the worker poller exactly.
- **Validation:** `pytest tests/test_jira_webhook.py -v` + full backend suite for
  the touched modules.
- **Commit:** `feat(jira-webhook): real-time inbound receiver (fail-closed HMAC)`

## 4. Testing Strategy
- Backend: model + migration + routes + receiver (`test_jira_webhook*.py`).
- Consistency: assert webhook reconcile and poller produce identical
  `workflow_status` + one event for the same input (shared `status_sync_core`).
- Manual: configure a Jira Cloud webhook with the secret pointing at
  `/api/v1/webhooks/jira/inbound`; move an issue; confirm feedback flips in
  seconds + one timeline event; disable webhook → poll still works.

## 5. Environment & Secrets
- `LLM_ENCRYPTION_KEY` (existing Fernet key) required; missing → fail-closed (R6),
  no plaintext secret. Inbound URL must be reachable from Jira Cloud (documented
  for self-hosters; localhost needs a tunnel).

## 6. Edge Cases & Error Handling
- Fail-closed on any signature/secret problem (401). Unknown org → 401.
- Non-status events / issues with no linked feedback → 200 no-op.
- Multiple links for one feedback → most-advanced category wins (poller parity).
- Multi-tenancy: reconcile strictly org-scoped via the resolved integration.
- Poll remains the fallback for missed deliveries.
