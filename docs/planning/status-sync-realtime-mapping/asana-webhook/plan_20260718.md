### Phase 3 — Receiver: handshake + events (RED → GREEN → REFACTOR)
- **Files:** new `services/backend-api/src/api/routes/asana_webhook.py`;
  register in `api/main.py`; new
  `services/backend-api/src/services/asana_status_reconcile.py` (mirror Zendesk
  port, reuse race-safe apply); `tests/test_asana_webhook.py`.
- **Steps:**
  1. RED (`test_asana_webhook.py`, mirror `test_zendesk_webhook.py`):
     - **Handshake:** first POST with `X-Hook-Secret` and no stored secret →
       persist encrypted + echo the same value in response `X-Hook-Secret`
       header, 200, no reconcile.
     - **Event, valid `X-Hook-Signature`** (hex HMAC-SHA256 over raw body) →
       processed; bad/missing signature → 401; missing stored secret post-handshake
       → 401 (fail-closed).
     - Completion change on a linked task → `workflow_status` updated per merged
       mapping; exactly one event `metadata={"source":"asana",...}`.
     - Stale/duplicate delivery → zero events (race guard).
  2. GREEN: implement `POST /api/v1/webhooks/asana/inbound`:
     - If `X-Hook-Secret` header present and integration has no secret → store +
       echo header, return 200 (handshake branch, I/O-free).
     - Else verify `X-Hook-Signature`; resolve integration by `webhook_gid`
       (payload) / secret; 401 on failure.
     - For each event with a changed task resource, fetch `completed`
       (`asana_client.get_task`) → `asana_category(completed)`; reconcile via port:
       most-advanced category → `resolve_target_status(cat, integ.status_mapping)`
       → race-safe apply → upsert last-observed; `db.commit()`.
     - Return 200.
  3. REFACTOR: keep reconcile mirroring the worker poller.
- **Validation:** `pytest tests/test_asana_webhook.py -v` + touched-module suite.
- **Commit:** `feat(asana-webhook): inbound receiver with handshake + HMAC (fail-closed)`

## 4. Testing Strategy
- Backend: model + migration + routes + receiver, with the **handshake** path
  explicitly tested (echo header, 200, secret persisted).
- Consistency: webhook reconcile and poller yield identical result for the same
  `completed` change.
- Manual: enable webhook, complete a linked Asana task, confirm feedback flips in
  seconds + one event; disable → poll fallback works.

## 5. Environment & Secrets
- `LLM_ENCRYPTION_KEY` required (fail-closed if missing, R6). Inbound URL must be
  publicly reachable for Asana to deliver (documented; localhost needs a tunnel).
  Handshake response must be fast (<~10s) — keep it I/O-free.

## 6. Edge Cases & Error Handling
- Re-handshake (Asana re-establishes webhook): idempotent secret update.
- Fail-closed on signature/secret problems (401). Unknown webhook gid → 401.
- Events for tasks with no linked feedback → 200 no-op.
- Category-level only (completed/new); section-name granularity explicitly out of
  scope. Multi-tenancy: reconcile org-scoped via resolved integration.
- Poll remains fallback.
