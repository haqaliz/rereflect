## 6. Edge cases & error handling

- **Second head appears before merge (R-DM2).** Re-run `alembic heads` immediately **before merging**, not
  only at authoring. If another branch landed a revision, rebase `down_revision` onto the new head and
  re-run the round-trip. Never merge a revision that forks the graph.
- **`status` accepts junk at the DB layer — by design.** No CHECK (house convention). The routes aspect
  imports `CHURN_SUGGESTION_STATUSES` rather than redeclaring it. `test_no_db_check_on_status` pins this as
  intentional, not an oversight.
- **Unique collision is the idempotency guarantee, not a bug.** Re-harvesting the same opportunity must
  raise `IntegrityError`; the harvester aspect decides how to swallow it. No `ON CONFLICT` here.
- **`ON DELETE SET NULL` on `churn_event_id` orphans provenance** when an event is deleted; the suggestion
  keeps `status='confirmed'`. Accepted per `prd.md:302` / R-DM6 — do not "improve" to CASCADE.
- **`evidence` is unbounded JSON** (R-DM5). Out of scope: nullable JSON; the harvester owns payload/caps.
- **Parity test session poisoning.** Skipping the `finally` `sys.modules` restore makes every subsequent
  test import the wrong `src`. Copy the block whole; do not refactor it here.
