### Phase 3 — Harvester service (idempotency, suppression, cap)
**Goal.** Write pending rows once, never twice, never an event.
**Prerequisites.** Phases 1–2.
**Files.** `src/services/churn_suggestion_harvester.py`, `tests/test_churn_suggestion_harvester.py`.
**Signature** (D1 applied):
```python
PER_RUN_SUGGESTION_CAP = 200
def harvest_org_suggestions(org_id, db, client, *, provider, renewal_set, known_emails,
                            company_ids, cap=PER_RUN_SUGGESTION_CAP) -> dict:
    # {scanned, suggested, skipped_existing, denied, dropped_by_cap, status}
```
**Steps.**
1. RED: hand-written **Fake client class** (not MagicMock — AC 9 forbids `patch`) + file-local
   SQLite `db` fixture. N candidates → N `pending` rows.
2. RED (**AC 4**): second run inserts zero — `scanned > 0`, `suggested == 0`,
   `skipped_existing == N`, row `id`s unchanged (no update-on-reharvest).
3. RED (AC 5): existing `rejected` not re-suggested; `confirmed` not duplicated; customer with an
   active `CustomerChurnEvent` skipped; **recovered** (`recovered_at` set) **not** skipped.
4. RED (AC 6): 250 candidates, `cap=200` → 200 rows, `dropped_by_cap == 50`, **WARNING** naming
   org_id/provider/cap/dropped (`caplog`). No silent caps.
5. RED (**AC 8**): `db.query(CustomerChurnEvent).count()` before == after on the happy path.
6. RED: a raising Fake client → `{"status":"error", ...}`, logged, not propagated.
7. GREEN: iterate `sorted(company_ids.items())` for determinism; memoize the per-company fetch
   (`salesforce_sync.py:251-252`). Per candidate: adapt → `decide_suggestion` → deny increments
   `denied` (counters, not rows). On suggest: pre-check `(org, provider, external_opportunity_id)`
   → skip if a row exists in **any** status; else `db.add` + `db.flush()` in `try/except
   IntegrityError` inside `db.begin_nested()` (SAVEPOINT) → `skipped_existing += 1`. No `ON
   CONFLICT` (SQLite-safe, per `_upsert_enrichment`); the DB UNIQUE is the real guarantee. Caller
   owns the transaction — **no `db.commit()`**. Active-event check reimplemented locally (`org +
   email + recovered_at IS NULL`, mirroring `routes/churn_events.py:74-88`) — cannot import backend.
   **Never** construct or write `CustomerChurnEvent` — review-queue's job on human confirm.
**Validation.** `pytest tests/test_churn_suggestion_harvester.py -q` green; zero Celery/httpx/`patch`.
**Commit.** `feat(harvester-core): idempotent, capped churn suggestion harvester`

## 6. Edge cases & error handling

| Case | Behaviour |
|---|---|
| `churn_labels_enabled=False` (default) | Skip; zero rows; result dict identical to baseline (AC 7). |
| Enabled, config empty/absent/`None` | Skip — `discriminator_not_configured`. Default-deny survives the toggle. |
| `pipeline`/`Type` None/`""`/whitespace | `no_discriminator` (R5). Silence, not a false label. |
| Near-match (`"renewal"` vs `"Renewal"`) | Deny. Exact match only — no regex, no normalization (M2). |
| Email not in `known_emails` | `unknown_customer`. |
| Missing/unparseable close date or id | Adapter → `None`; dropped, counted in `denied`. |
| Row exists in **any** status (incl. `rejected`) | Skip, never update. Re-harvest is a no-op, not a refresh. |
| Race → `IntegrityError` | Caught, SAVEPOINT rollback, `skipped_existing += 1`. DB UNIQUE is the guarantee. |
| Active `CustomerChurnEvent` | Skip. **Recovered** (`recovered_at` set) → not skipped. |
| Two contacts share a company/account | Fetch memoized once; shared deal dedups on the UNIQUE pre-check. |
| >`cap` candidates | Insert `cap`, count the rest, **WARNING** with `dropped_by_cap`. No silent caps. |
| Any harvest exception | Caught at the `_sync_org` seam, `logger.exception`, `status="error"`; enrichment + `last_sync_status='success'` intact (R4). |
| Writing `customer_churn_events` | **Forbidden.** count-before == count-after (AC 8). |
