## 6. Edge cases & error handling

| Case | Required behaviour | AC |
|---|---|---|
| Malformed / empty / 14-char SF account id | `SalesforceQueryError`, **zero** HTTP calls | AC-3 |
| SF `Type` absent from describe, or present with empty picklist | `[]` (not `None`, not raise) | AC-6 |
| SF opportunity record with `Type: None` | returned **verbatim**; client never interprets it | AC-5/R5 |
| HubSpot deal with `pipeline` missing or `None` | returned **unaltered**; never filtered | AC-5 |
| HubSpot company with no deal associations / 404 on assoc | `[]` from both accessors | §7 |
| `list_deal_pipelines` 404 | `[]` | AC-6 |
| 429 with `Retry-After: 7` | `time.sleep(7)` → transient error | AC-7 |
| 429 with no `Retry-After` | defaults to `10` | AC-7 |
| 500 / 503 | transient error, **no sleep** | AC-7 |
| 403 (narrow-scoped connected app / missing HubSpot scope) | `SalesforceScopeError` / `HubSpotScopeError`; classify only, never disable | AC-7 |
| SF 401 | `_refresh()` **once**, then `SalesforceAuthError` | AC-7 |
| Any error path | never touches `is_active`, never `self.retry` (callers own retry), never logs a token | AC-7/AC-8 |

**No new exception types.** Reuse the six shipped ones (`HubSpotTransientError`, `HubSpotScopeError`, `HubSpotNotFoundError`, `SalesforceTransientError`, `SalesforceScopeError`, `SalesforceQueryError`, `SalesforceAuthError`).
