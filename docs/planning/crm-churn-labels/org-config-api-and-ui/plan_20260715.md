## 6. Edge cases & error handling

| Case | Behaviour |
|---|---|
| `{enabled: true}`, empty/absent renewal list | **200**, no CRM call, config persisted empty. **Never invent "all pipelines"** — that injects the exact false labels the PRD exists to prevent. Card warns (spec §13). |
| `{enabled: false}`, no config | 200; disables; **leaves saved config intact** (`if payload.config is not None` — `jira_integration.py:574`). |
| Saved config holds an id later deleted in HubSpot | PATCH on an untouched id **never 422s** (only payload ids are validated). Picker shows it flagged; never auto-pruned (R-C). |
| CRM 401/403 | 502 `reason=missing_read_scope`, surfaced. **`is_active` untouched — never auto-disconnect.** |
| CRM 429/5xx/network | 502 `reason=options_fetch_failed` (assert **not** 500). |
| `Opportunity.Type` absent / no pipelines | `([], None)` → 200 `{options: []}`; card says so rather than render a dead picker (R-D). |
| No integration row / row inactive | PATCH: 404 / 400. `/options`: 404 (needs a live token). Asymmetry is spec §4-5 — intentional. |
| Card mount while CRM is down | Toggle + stats still render; only the picker degrades (R-A). |
| Config with unknown key / string value / non-string member | 422 each, row unchanged (validate before mutate). |
