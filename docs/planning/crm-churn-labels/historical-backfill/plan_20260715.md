### Deltas found against spec/PRD (decisions taken)

1. **Alembic head.** PRD says chain off `e6f7a8b9c0d1`; spec §"Dependencies" repeats a **two-head fork** claim
   that the PRD itself withdrew (R6). **LIVE `alembic heads` in this worktree ⇒ `f7a8b9c0d1e2` (single head).**
   `e6f7a8b9c0d1` is *master's* head; this branch already added `54cc807` → `f7a8b9c0d1e2`. **We chain off
   `f7a8b9c0d1e2`.** There is no fork to resolve. Re-run `alembic heads` at write time; never grep the files.
2. **"Org-wide queries" (spec §4) is not buildable as written for HubSpot.** The adapters take `contact_email`
   from the caller, and the email↔company link only exists on the *contact* record
   (`props.associatedcompanyid` / `Contact.AccountId`). An org-wide HubSpot deal search returns deals with no
   email. `crm_enrichment` has `hubspot_company_id` but **no `salesforce_account_id`** — so the map cannot be
   read from the DB either. **Decision: backfill builds `company_ids` from `client.list_contacts()` exactly as
   `_sync_org` does (hubspot_sync.py:261-272, salesforce_sync.py:304-326), then pages per company with a window
   floor.** Same shape as the shipped harvester; "window-bounded" is honored, "org-wide" is not literally
   achievable without a new association-resolution path. Recorded, not silently dropped.
3. **Abort granularity.** Spec §1 says "abort check between pages". Paging lives *inside* the shipped clients
   (`query()`, `_fetch_deals_for_company`) and forking them to expose page boundaries is R4 risk for zero
   benefit. **Decision: the abort check runs between *fetch units* (companies)** — the finest boundary the
   backfill controls. Documented in the task docstring.
4. **Columns.** `last_harvest_*` do not exist (read via `getattr(..., None)`, routes:242-243). They are
   harvester-core's should-have, **not ours** — we do not add them. We **do** add the four `backfill_*` columns
   spec §6 requires. **→ Yes, this plan adds a migration.**

## 6. Edge cases & error handling

- **Empty/absent renewal set** → default-deny; task returns early, endpoint 400. Never an "all pipelines" default.
- **`months` at bounds** → 1 and 60 valid; 0 and 61 → 422 at the schema.
- **Null / unknown `Type` / `pipeline`** → no suggestion (PRD R5), identical to forward harvest.
- **Deal with no id or unparseable `closedate`** → adapter returns `None` → counted `denied`, never a row.
- **Customer already has an active `CustomerChurnEvent`** → skipped (`_has_active_churn_event`).
- **Concurrent insert race** → `begin_nested` + `IntegrityError` → `skipped_existing`; the DB UNIQUE is the guarantee.
- **Worker restart mid-run** → status stays `running`; re-running is safe (AC-5). Progress committed per unit.
- **Cancel on a dead run** → status is advisory; a stale `running` is cleared by the next successful trigger.
- **SF daily-limit near-exhaustion** → shipped `query()` stops paging and warns; surfaced via `dropped_by_cap`/log, not swallowed.
- **429 / 5xx / 403 / SF 401** → shipped taxonomy verbatim; `is_active` never touched.
- **One CRM per org** guard is application-level with a documented TOCTOU race — the core stays provider-agnostic.
