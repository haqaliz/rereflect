## 6. Edge cases & error handling

- **Empty org** — `trainable = 0`, `pending = 0`, `ready False`. `or 0` guards each `.scalar()`.
- **All-auto org** — `total = N`, `trainable = 0`, `ready False`. The honest case the bug hid.
- **Recovered + trainable source** — counts in `total`, its bucket, **and** `trainable` (AC-4).
- **Suggestion with no event** — pending only; never touches `total`/`trainable`/`ready` (AC-5).
- **Confirmed suggestion** — counts via its `CustomerChurnEvent(source='manual')`, not itself (AC-6).
- **The number will go down** — an operator at "500/500 Ready" may now see "499/500 Not Yet Ready". That is
  the fix working. M8 (`docs-and-tracking`) must say so plainly rather than ship a silent correction to a
  trust surface — out of scope here; hand off explicitly.
- **Unknown/future source values** — anything `!= 'auto_suggested'` is trainable, matching the four filters
  exactly. Do **not** substitute an allowlist; that re-introduces the disagreement.
- **`/customers` link** — a plain `next/link`; the review view lands in wave 3 (delta 1).
