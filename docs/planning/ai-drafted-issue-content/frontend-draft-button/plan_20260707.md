## Phase 1 — API client (`lib/api/issueDraft.ts`)
- **Goal:** typed client for the draft endpoint.
- **Files:** create `lib/api/issueDraft.ts` + `lib/api/__tests__/issueDraft.test.ts`.
- **Steps (TDD):**
  1. `export interface IssueDraft { title: string; body: string }`.
  2. `export async function draftIssueContent(feedbackId: string, target: "jira" | "asana"): Promise<IssueDraft>`
     → `POST /api/v1/feedback/${feedbackId}/issue-draft` with `{ target }`, auth header per existing clients.
  3. Throw a typed error carrying the HTTP status (so the caller can distinguish 409 vs other) — mirror how the
     other `lib/api` clients surface non-2xx.
- **Validation:** `npm run test -- issueDraft` — success parse + non-2xx throw (incl. 409).
- **Commit:** `feat(issue-draft-ui): lib/api/issueDraft client`

## Phase 2 — Wizard button + populate + overwrite-confirm
- **Goal:** the "✨ Draft with AI" button in both branches, populating fields with confirm-if-edited.
- **Prereq:** Phase 1.
- **Files:** modify `create-issue/page.tsx`; test `__tests__/createIssueDraft.test.tsx`.
- **Steps (TDD):**
  1. Add state: `draftingJira`, `draftingAsana` booleans (mirror `generating`); `aiConfigured` boolean.
  2. **T2 gate:** on mount, call `aiSettingsAPI.get()` (or `listKeys()`); set `aiConfigured` = LLM present
     (cloud key **or** local provider+base_url). Render each draft button only when `aiConfigured`.
  3. **Seed tracking for M8:** capture the auto-seeded defaults (already computed at `:195-209`) into refs
     (`seededJiraSummary`, etc.) so "has the user edited?" = current value !== seeded default.
  4. **Handler** `handleDraft(target)`:
     - set drafting=true; `try { const d = await draftIssueContent(feedbackId, target) }`.
     - **Overwrite rule:** if the relevant fields are empty or equal their seeded defaults → set directly. If
       edited → `window.confirm("Replace your text with the AI draft?")` (or the app's confirm dialog if one is
       standard); on confirm set, else no-op.
     - Jira: `setJiraForm(prev => ({ ...prev, summary: d.title, description: d.body }))`.
       Asana: `setAsanaForm(prev => ({ ...prev, name: d.title, notes: d.body }))`.
     - **Never** call `handleJiraSubmit`/`handleAsanaSubmit`.
     - `catch`: `toast.error(...)` (for a thrown 409, show the honest "configure an AI model" message); fields
       untouched. `finally`: drafting=false.
  5. **Button UI:** `Sparkles` icon + label "Draft with AI"; while drafting show `Loader2` spinner + disabled;
     also disabled if a create is in progress. Place next to the Summary/Name field in each branch.
- **Validation:** `npm run test -- createIssueDraft` and `npm run lint`. Cover: button hidden when
  `aiConfigured=false`; click populates fields + no create call; edited-field confirm branch (accept + cancel);
  error toast + fields untouched; disabled-while-drafting.
- **Commit:** `feat(issue-draft-ui): Draft-with-AI button in Jira+Asana wizard branches (hidden when no LLM)`

## Testing strategy
- **Vitest** with the endpoint + `aiSettingsAPI` mocked. Assert **no** call to the create APIs when drafting.
- Commands: `cd services/frontend-web && npm run test -- issueDraft createIssueDraft && npm run lint`.

## Edge cases & error handling
- No LLM (T2): button hidden; even if forced, 409 → toast, no field change.
- Regenerate: re-click re-runs handler (confirm rule reapplies since fields now differ from seed).
- Concurrency: button disabled while `drafting` true.
- CSS: use theme variables / existing button variants — no hardcoded colors.
