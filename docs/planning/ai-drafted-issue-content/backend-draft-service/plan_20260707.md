## Phase 1 ‚Äî Drafting service (`issue_drafter.py`)

- **Goal:** a pure, testable async function that produces `{title, body}` from a feedback item.
- **Prereq:** none.
- **Files:** create `src/services/issue_drafter.py`; test `tests/test_issue_draft.py` (service tests).
- **Signature:**
  ```python
  async def draft_issue_content(
      feedback: FeedbackItem,
      org: Organization,
      target: str,                 # "jira" | "asana"
      db: Session,
      tone: Optional[str] = None,
  ) -> dict:                       # {"title": str, "body": str}
  ```
- **Steps (TDD, write the failing test first each step):**
  1. **Gate:** call `resolve_generation_llm(org.id, db)`. If `not is_configured` ‚Üí raise a typed
     `LLMNotConfiguredError` (new, in this module). Test: asserts raise + that no LLM client is constructed.
  2. **Prompt build:** system prompt instructs "you write a concise work-tracker {target} title + body from
     customer feedback"; resolve tone = `tone or org.default_tone or "professional"`; inject
     `org.brand_voice` / `org.product_name_display` like `response_generator`. **E1 hardening:** put the
     feedback text inside a clearly delimited block (e.g. `<feedback>‚Ä¶</feedback>`) and state it is untrusted
     data to summarize, not instructions. Include structured signals: `sentiment_label`, `tags`, `is_urgent`,
     `pain_point_*` / `feature_request_*` when present. **E2:** cap feedback text fed to the model (e.g.
     `feedback.text[:4000]`) ‚Äî name the constant `MAX_FEEDBACK_CHARS`.
  3. **Output contract:** instruct the model to return **strict JSON** `{"title": "...", "body": "..."}`.
     Request `response_format={"type":"json_object"}` when provider supports it; always parse defensively.
  4. **LLM call:** mirror `call_llm_stream`'s client construction but `stream=False`:
     `client = openai.AsyncOpenAI(api_key=cfg.api_key or "ollama", base_url=cfg.base_url, timeout=60.0)` when
     `base_url` else `openai.AsyncOpenAI(api_key=cfg.api_key or "", timeout=60.0)`; then
     `resp = await client.chat.completions.create(model=cfg.model, messages=..., stream=False, temperature=0.7, max_tokens=700, response_format=...)`.
     Extract `resp.choices[0].message.content`.
  5. **Parse + validate (E3):** `json.loads`; require non-empty `title` and `body` (strings). On
     parse failure / empty ‚Üí raise `IssueDraftError`. Trim `title` to ‚â§255 (create-route guard is final).
  6. **Usage log (M6):** write `LLMUsageLog(organization_id=org.id, provider=cfg.provider, model=cfg.model,
     task_type="issue_draft", prompt_tokens=resp.usage.prompt_tokens, completion_tokens=..., total_tokens=...,
     estimated_cost_cents=<pricing or 0.0>, latency_ms=<measured>)`. For cost: reuse the existing pricing
     helper if one is trivially importable from `llm_model_price`; otherwise `0.0` (local models are free).
     Wrap in try/except so a logging failure never fails the draft.
  7. Return `{"title": title, "body": body}`.
- **Validation:** `pytest tests/test_issue_draft.py -v` (service-level tests green). Mock the OpenAI client via
  `AsyncMock` (patch `src.services.issue_drafter.openai.AsyncOpenAI`), returning a stub with
  `.choices[0].message.content` JSON and `.usage`.
- **Commit:** `feat(issue-draft): issue_drafter service ‚Äî LLM gate, hardened prompt, {title,body} parse + usage log`

## Phase 2 ‚Äî Route (`feedback_issue_draft.py`)

- **Goal:** expose the service at `POST /api/v1/feedback/{feedback_id}/issue-draft`.
- **Prereq:** Phase 1.
- **Files:** create `src/api/routes/feedback_issue_draft.py`; extend `tests/test_issue_draft.py` with route tests;
  modify `src/api/main.py` (register router).
- **Steps:**
  1. `router = APIRouter(prefix="/api/v1/feedback", tags=["feedback"])` (mirror `feedback_responses.py`).
  2. Pydantic `IssueDraftRequest { target: Literal["jira","asana"]; tone: Optional[str] = None }` (extra=forbid);
     `IssueDraftResponse { title: str; body: str }`.
  3. Handler deps: `get_current_org`, `get_current_user`, `require_admin_or_owner` (match create routes).
  4. Load `FeedbackItem` by `feedback_id` **and** `organization_id == org.id`; not found ‚Üí **404**.
  5. `try: draft = await draft_issue_content(feedback, org, payload.target, db, payload.tone)`
     - `except LLMNotConfiguredError:` ‚Üí **409** `{"detail": "No AI model configured. Configure a provider in AI Settings or set a local LLM to use AI drafting."}`
     - `except IssueDraftError:` ‚Üí **502** `{"detail": "The AI model returned an unusable draft. Try again."}`
     - `except Exception:` (provider/timeout) ‚Üí **502** honest message; log server-side.
  6. Return `IssueDraftResponse`.
  7. Register in `src/api/main.py` next to the other feedback routers.
- **Validation:** `pytest tests/test_issue_draft.py -v` ‚Äî cover the 8 acceptance criteria (200 jira, 200 asana,
  409 unconfigured + no client call, 404 cross-org, 422 bad target, 403 member, 502 malformed, one usage row,
  E1 injection-as-data assertion on the built prompt). Mock the service (`AsyncMock` patch in the route module)
  for route tests; keep the real-service LLM-client-mock tests from Phase 1.
- **Commit:** `feat(issue-draft): POST /api/v1/feedback/{id}/issue-draft (409 when no LLM, 404 cross-org)`

- **Route:** the 8 acceptance criteria. Mock `draft_issue_content` with `AsyncMock`.
- **Command:** `cd services/backend-api && pytest tests/test_issue_draft.py -v`. Before commit, also run the
  neighbors that share the router registration: `pytest tests/test_response_generation.py tests/test_jira_issues.py -q`
  to confirm no regressions from the `main.py` change. (Repo caveat: run scoped test files, not the full suite ‚Äî
  a pre-existing `test_report_ws.py` segfault makes bare full-suite runs unreliable.)

## Edge cases & error handling
- 409 path must **not** construct an LLM client (assert in test).
- Multi-tenancy: feedback lookup filtered by `organization_id`; cross-org ‚Üí 404 (never 403, to avoid existence leak).
- E2 truncation constant; E3 malformed ‚Üí 502; usage-log failure swallowed (never fails the request).
- `target` validated by `Literal` ‚Üí 422 automatically.
