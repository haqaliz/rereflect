## 0. Scope & non-goals

**In scope (this aspect):**

1. `src/services/saml_provider.py` — a pure, route-free SAML provider service wrapping `python3-saml`
   (OneLogin): builds an SP-initiated **unsigned** AuthnRequest (HTTP-Redirect) and validates a returned
   SAML Response/Assertion to a trusted identity, delegating signature + condition checks to the library.
2. `src/models/saml_auth_request.py` — the DB-backed `InResponseTo`/replay store table.
3. `src/services/saml_replay.py` — the explicit `pending → consumed` state machine over that table
   (register on issue, race-safe conditional-consume on ACS, opportunistic expired-row cleanup).
4. One Alembic migration creating `saml_auth_requests` (chains **after** the config aspect's migration).
5. Unit tests: `tests/test_saml_provider.py`, `tests/test_saml_replay.py`.

**Out of scope (later aspects / explicit non-goals):**

- The FastAPI `/saml/login` and `/saml/callback` routes that call this service (`login-routes-and-identity`).
- IdP-initiated flow (no `InResponseTo`), SLO, encrypted assertions, signed AuthnRequests / SP private key.
- Adding `python3-saml`/`xmlsec` to `requirements.txt` + `Dockerfile` (that is `deps-and-docker`).
- The `saml_configs` model + CRUD + `users.saml_subject` (that is `config-model-and-crud`) — this aspect
  only **reads** a `SamlConfig` row via `SamlProvider.from_config`.

## 2. Design

### 2.1 `src/services/saml_provider.py`

Mirror `oidc_provider.py` in structure: module docstring stating the security posture, module-level
`_now()` shim, a typed error, and one class with `from_config`.

**Module-level:**

```python
_now() -> float                      # time.time() wrapper; monkeypatchable for skew/expiry tests (mirror OIDC)

SAML_ACS_PATH = "/api/v1/auth/saml/callback"          # ACS route the next aspect will mount
SAML_METADATA_PATH = "/api/v1/auth/saml/metadata"     # SP entityId basis
SAML_REQUEST_TTL_SECONDS = int(os.getenv("SAML_REQUEST_TTL_SECONDS", "600"))  # match STATE_TTL_SECONDS=600
CLOCK_SKEW_SECONDS = 60                                # ±60s tolerance (spec / R5)

# sso_error codes this feature emits. The provider raises the first five directly;
# `unsolicited`/`replay` are raised by the replay store at the ACS boundary (see 2.3),
# but the full set is defined here as the single source of truth.
SAML_ERROR_CODES = frozenset({
    "signature", "assertion", "audience", "recipient", "expired", "unsolicited", "replay",
})

class SamlValidationError(Exception):
    """Typed validation failure. `.code` is one of SAML_ERROR_CODES; the route maps it to ?sso_error=."""
    def __init__(self, code: str, detail: str = ""): ...   # stores self.code, self.detail
```

**`ValidatedAssertion`** — a frozen dataclass, the only thing `validate_response` returns:

```python
@dataclass(frozen=True)
class ValidatedAssertion:
    subject: str                 # SAML NameID (never None — null-subject is rejected before construction)
    email: Optional[str]         # extracted per the chain in 2.2; lowercased
    attributes: dict             # auth.get_attributes() — for the route/identity aspect
```

**`class SamlProvider`:**

- `__init__(self, *, sp_entity_id, acs_url, idp_entity_id, idp_sso_url, idp_x509_cert, email_attribute=None)`
  — plain fields, no I/O. `idp_x509_cert` is the operator-supplied **public** PEM (not decrypted — SAML
  signing certs are public, unlike the OIDC `client_secret`).

- `@classmethod from_config(cls, cfg) -> "SamlProvider"`:
  - Derive SP identifiers from `BACKEND_URL` (mirror `_backend_url()` in `auth.py` — replicate a private
    `_backend_url()` reading `os.getenv("BACKEND_URL", "http://localhost:8000")` rather than importing the
    route module, to keep the service route-free):
    - `sp_entity_id = f"{backend_url}{SAML_METADATA_PATH}"`
    - `acs_url = f"{backend_url}{SAML_ACS_PATH}"`
  - Copy `idp_entity_id`, `idp_sso_url`, `idp_x509_cert`, `email_attribute` from `cfg`.
  - **No decryption** (contrast with `OidcProvider.from_config` which decrypts `client_secret`).

- `_build_settings(self) -> dict` — the OneLogin settings dict handed to `OneLogin_Saml2_Settings` /
  `OneLogin_Saml2_Auth`:
  ```python
  {
    "strict": True,                       # enforce all conditions; never lenient
    "debug": False,
    "sp": {
      "entityId": self.sp_entity_id,
      "assertionConsumerService": {"url": self.acs_url,
        "binding": "urn:oasis:names:tc:SAML:2.0:bindings:HTTP-POST"},
      "NameIDFormat": "urn:oasis:names:tc:SAML:1.1:nameid-format:unspecified",
    },
    "idp": {
      "entityId": self.idp_entity_id,
      "singleSignOnService": {"url": self.idp_sso_url,
        "binding": "urn:oasis:names:tc:SAML:2.0:bindings:HTTP-Redirect"},
      "x509cert": <PEM body, headers stripped>,   # OneLogin wants base64 body; strip BEGIN/END lines
    },
    "security": {
      "wantAssertionsSigned": True,       # REQUIRED signed assertion (M8/R2) — reject unsigned/response-only-signed
      "wantMessagesSigned": False,        # response-level sig optional; assertion sig is the trust anchor
      "wantAssertionsEncrypted": False,   # encrypted assertions out of scope
      "wantNameId": True,                 # subject must be present (backs null-subject rejection)
      "wantNameIdEncrypted": False,
      "authnRequestsSigned": False,       # UNSIGNED SP request (no SP key in slice 1)
      "rejectUnsolicitedResponsesWithInResponseTo": False,  # we ALWAYS pass request_id; store owns unsolicited
      "requestedAuthnContext": False,
      "signatureAlgorithm": "http://www.w3.org/2001/04/xmldsig-more#rsa-sha256",
      "digestAlgorithm": "http://www.w3.org/2001/04/xmlenc#sha256",
    },
  }
  ```
  Cache the built `dict` per instance (cheap; no module-level cache needed — unlike OIDC discovery/JWKS
  there is no network fetch, so the OIDC `_discovery_cache`/`_jwks_cache` pattern is **not** carried over).

- `_require_https_public(self, url: str) -> str` — the SSRF gate, mirroring the OIDC method's **shape**
  but simplified (there is no discovery document and no issuer-containment case, so drop the
  `require_issuer_host` branch):
  1. `urlparse(url)`; reject if `scheme != "https"` → `SsrfError`.
  2. `assert_host_not_ssrf(parsed.hostname)` (loopback/RFC1918/link-local reject).
  3. return the host. Raises `SsrfError` on any failure.
  Called from `build_authn_request` against `self.idp_sso_url` (defense-in-depth; the config aspect also
  SSRF-validates on save, but the provider re-gates on use — same discipline as OIDC re-gating discovered
  URLs every call).

- `build_authn_request(self, relay_state: str) -> tuple[str, str]`:
  1. `self._require_https_public(self.idp_sso_url)`.
  2. Build the OneLogin `request_data` dict the library expects for URL construction:
     ```python
     backend = urlparse(self._backend_url())
     request_data = {"https": "on", "http_host": backend.netloc,
                     "script_name": SAML_ACS_PATH, "get_data": {}, "post_data": {}}
     ```
  3. `auth = OneLogin_Saml2_Auth(request_data, old_settings=self._build_settings())`
  4. `redirect_url = auth.login(return_to=relay_state)` — HTTP-Redirect binding, **unsigned** (since
     `authnRequestsSigned=False`). `login()` returns the fully-built IdP redirect URL (SAMLRequest +
     RelayState in the query string).
  5. `request_id = auth.get_last_request_id()` — the generated AuthnRequest `ID`; the caller persists it
     via the replay store (2.3) so the ACS can match `InResponseTo`.
  6. `return redirect_url, request_id`.
  - `relay_state` is opaque to this method (the route builds an HMAC-signed RelayState via the reused
    `sign_state`/`hash_nonce`; provider just threads it into `return_to`).

- `validate_response(self, saml_response_b64: str, *, expected_in_response_to: str, acs_url: str) -> ValidatedAssertion`:
  1. Build `request_data` for the POST binding so the library's Destination/Recipient check compares
     against the **real** ACS URL:
     ```python
     acs = urlparse(acs_url)
     request_data = {"https": "on", "http_host": acs.netloc, "script_name": acs.path,
                     "get_data": {}, "post_data": {"SAMLResponse": saml_response_b64}}
     ```
  2. `auth = OneLogin_Saml2_Auth(request_data, old_settings=self._build_settings())`.
  3. `auth.process_response(request_id=expected_in_response_to)` — **the single delegated validation
     call.** Passing `request_id` makes the library enforce `Response/@InResponseTo == expected`. With
     `strict=True` + `wantAssertionsSigned=True` this one call enforces: XML signature over the assertion
     against `idp.x509cert`, `Issuer == idp.entityId`, `Audience == sp.entityId`,
     `Destination`/`Recipient == acs_url`, and `NotBefore`/`NotOnOrAfter`.
  4. `errors = auth.get_errors()`. If non-empty, map via `auth.get_last_error_reason()` (case-insensitive
     substring match) to a `SamlValidationError(code)` — see the mapping table in 2.2. **Never** fall
     through to reading identity out of a failed response.
  5. On success, extract identity **only from the library's validated getters** — never re-parse the raw
     XML for subject/email (that is exactly the XSW hole, R2):
     - `name_id = auth.get_nameid()`; `name_id_format = auth.get_nameid_format()`;
       `attributes = auth.get_attributes()`.
     - **Null-subject rejection (kept from OIDC):** if `not name_id` → `SamlValidationError("assertion")`.
       Without it, a later `User.saml_subject == None` query would compile to `WHERE saml_subject IS NULL`
       and match every non-SAML user (the account-takeover the OIDC code guards against at L500-507).
     - Email via the chain in 2.2.
     - **±60s clock skew supplement (see 2.4)** — apply here if `process_response` rejected purely on a
       timestamp condition.
  6. `return ValidatedAssertion(subject=name_id, email=email, attributes=attributes)`.

### 2.2 Error mapping & email extraction

**Error-reason → code** (matched against `auth.get_last_error_reason()`, first match wins; default
`assertion` so nothing fails open to success):

| reason contains (case-insensitive) | `SamlValidationError.code` |
|---|---|
| `signature` (verify failed / invalid) | `signature` |
| `not signed`, `unsigned`, `no signature`, `wantAssertionsSigned` | `assertion` |
| `audience` | `audience` |
| `recipient`, `destination` | `recipient` |
| `notonorafter`, `notbefore`, `expired`, `timestamp`, `is not yet valid`, `is no longer valid` | `expired` |
| `inresponseto` (mismatch/unknown at response level) | `unsolicited` |
| `issuer` | `assertion` (no dedicated `issuer` sso_error code exists — folds into `assertion`) |
| anything else / no reason | `assertion` |

Note `python3-saml` aggregates errors (e.g. `["invalid_response"]`) with the human detail in
`get_last_error_reason()`; the map keys off the reason string. Confirm exact phrasings against the
installed lib version during implementation and adjust substrings — **flag as a small iteration point.**

**Email extraction chain** (first hit wins; result lowercased; `None` if nothing matches — the route,
not the provider, decides whether a missing email is fatal, mirroring OIDC's `unverified`):

1. If `name_id_format` ends with `emailAddress`
   (`urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress`) **and** `name_id` contains `@` → use
   `name_id`.
2. Else if `self.email_attribute` is set and present in `attributes` → first value of that attribute.
3. Else default chain, first present of:
   `email` → `urn:oid:0.9.2342.19200300.100.1.3` → `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress`.

`attributes` values from `get_attributes()` are lists; take `[0]`.

### 2.3 Replay store — `src/models/saml_auth_request.py` + `src/services/saml_replay.py`

**Model `SamlAuthRequest` → table `saml_auth_requests`** (mirror the SQLAlchemy column style of
`oidc_config.py`):

| Column | Type | Notes |
|---|---|---|
| `request_id` | `String(255)` PK | the AuthnRequest `ID` (`_<uuid>`); primary key ⇒ implicitly unique |
| `organization_id` | `Integer` FK→`organizations.id` CASCADE, indexed | scope + FK cleanup |
| `created_at` | `DateTime` default `utcnow` | issue time |
| `expires_at` | `DateTime` not null, indexed | `created_at + SAML_REQUEST_TTL_SECONDS`; drives expiry + cleanup |
| `consumed_at` | `DateTime` nullable | set once, at first valid ACS use |

- `__table_args__`: `Index("ix_saml_auth_requests_org_id", "organization_id")`,
  `Index("ix_saml_auth_requests_expires_at", "expires_at")`.
- Export from `src/models/__init__.py` (add the import near line 62 and the name near line 135, beside
  `OidcConfig`).
- Use timezone-naive `datetime.utcnow` to match the rest of the codebase (`oidc_config.py`,
  `auth.py`).

**State machine** (explicit, gap 1):

| State | Set when | ACS outcome via `consume_request` |
|---|---|---|
| **pending** | row inserted at `/saml/login` (issue) | — |
| **consumed** | `consumed_at` set by the conditional UPDATE on first valid use | `OK` (first use) |
| replay | row already has `consumed_at` | `REPLAY` → `?sso_error=replay` |
| unsolicited | no row for that `request_id` | `UNSOLICITED` → `?sso_error=unsolicited` |
| expired | `expires_at < now` (still pending) | `EXPIRED` → `?sso_error=expired` |

**`src/services/saml_replay.py`** — thin, session-in / enum-out (no route imports):

```python
class ConsumeOutcome(str, Enum):
    OK = "ok"; REPLAY = "replay"; UNSOLICITED = "unsolicited"; EXPIRED = "expired"

def register_request(db, *, request_id, organization_id, ttl_seconds=SAML_REQUEST_TTL_SECONDS) -> None:
    # opportunistic bounded cleanup FIRST, then insert the pending row.
    _cleanup_expired(db)                                   # DELETE ... WHERE expires_at < :now  LIMIT-bounded
    now = _now()
    db.add(SamlAuthRequest(request_id=request_id, organization_id=organization_id,
                           created_at=now, expires_at=now + timedelta(seconds=ttl_seconds)))
    db.commit()

def consume_request(db, *, request_id) -> ConsumeOutcome:
    now = _now()
    # RACE-SAFE one-time consume: only the first caller flips consumed_at.
    result = db.execute(
        update(SamlAuthRequest)
        .where(SamlAuthRequest.request_id == request_id,
               SamlAuthRequest.consumed_at.is_(None),
               SamlAuthRequest.expires_at >= now)
        .values(consumed_at=now)
    )
    db.commit()
    if result.rowcount == 1:
        return ConsumeOutcome.OK
    # Disambiguate the miss:
    row = db.query(SamlAuthRequest).filter_by(request_id=request_id).first()
    if row is None:            return ConsumeOutcome.UNSOLICITED
    if row.consumed_at is not None: return ConsumeOutcome.REPLAY
    if row.expires_at < now:   return ConsumeOutcome.EXPIRED
    return ConsumeOutcome.UNSOLICITED     # defensive fallthrough
```

- `_now()` here returns a `datetime` (naive UTC) — either a local shim or reuse the provider's; keep one
  monkeypatch surface for the expiry tests. (The provider's `_now()` returns epoch seconds; the store
  works in `datetime`. Use **two clearly-named shims** — `saml_replay._now()` returning `datetime.utcnow()`
  — rather than sharing, to avoid unit confusion.)
- The conditional `UPDATE ... WHERE consumed_at IS NULL AND expires_at >= now` is the exactly-once
  guarantee: a duplicate/polled ACS POST re-runs the UPDATE, matches 0 rows, and resolves to `REPLAY`.
- `_cleanup_expired` is best-effort and bounded (e.g. delete rows with `expires_at < now`, optionally cap
  via a subquery `LIMIT`); a failure here must **not** block a login — wrap in try/except and log. No
  Celery beat (spec: opportunistic only).
- Note (documented non-goal): consumed **assertion-ID** tracking is not added — SP-initiated always
  carries `InResponseTo`, so the request-id store fully covers replay for slice 1.

### 2.4 Clock-skew handling (±60s) — known wrinkle, iteration checkpoint

`python3-saml`'s `process_response` validates `NotBefore`/`NotOnOrAfter` against its own
`OneLogin_Saml2_Utils.now()` with **no built-in skew tolerance** — the spec's ±60s (R5) is not a library
knob. Resolve one of two ways (recommend **A**):

- **Option A (recommended) — supplemental skew re-check, XSW-safe.** If `process_response` fails and the
  mapped code is `expired` (timestamp-only failure), re-read the `NotBefore`/`NotOnOrAfter` values from the
  **already-signature-validated** assertion object the library exposes (via its validated document /
  `OneLogin_Saml2_Utils.parse_SAML_to_time`), and accept iff `NotBefore - 60s <= now <= NotOnOrAfter + 60s`.
  Reading *timestamps* (not subject/email) after signature validation does not reintroduce XSW, because the
  bytes were signature-checked. Only apply the supplement to the timestamp branch; every other rejection
  stays hard.
- **Option B (fallback) — accept strict (0s) validation for slice 1** and record ±60s as a documented
  deviation. Cheaper, but the "not-yet-valid within skew passes" behavior is lost.

The `expired`/`not-yet-valid` **rejection** tests pass under either option (a fixture 5 min out of range is
rejected regardless). The skew *tolerance* test (a fixture 30s past `NotOnOrAfter` that should still pass)
only passes under A. **Flag this as an explicit iteration checkpoint** — pick A, verify against the
installed lib's validated-assertion surface, fall back to B only if that surface proves inaccessible.

     @NotOnOrAfter/AudienceRestriction`, `AttributeStatement`), then **sign the Assertion** with
     `OneLogin_Saml2_Utils.add_sign(xml, key_pem, cert_pem)` and base64-encode. Parametrize a
     `_make_response(**overrides)` factory so every rejection case is one template tweak.
   - Derive variants: **unsigned** (skip `add_sign`), **wrong-cert** (sign with a *different* keypair than
     the provider's configured cert), **XSW** (embed the signed assertion plus a second forged unsigned
     assertion — classic wrapping — and assert the library still yields the signed subject or rejects;
     confirm the forged subject never surfaces), **wrong-audience/recipient/issuer**, **expired**
     (`NotOnOrAfter` in the past), **not-yet-valid** (`NotBefore` in the future), **mismatched
     InResponseTo**, **null NameID**, and the three **email-source** variants (NameID emailAddress format /
     configured `email_attribute` / default-chain `urn:oid:...`/claims URI).
   - If in-test signing proves fiddly against the installed lib version, fall back to a **captured Keycloak
     SAML sample** re-signed with the throwaway key. **Call this out to the caller as a likely place to
     iterate.**
8. `tests/test_saml_provider.py` (RED, validation slice) — one test per variant above, each asserting the
   right `SamlValidationError.code` (per the 2.2 map) or, for the happy path, a `ValidatedAssertion` with
   the expected `subject`/`email`.
9. Implement `SamlProvider.validate_response` + the email chain + null-subject reject + the 2.2 error map
   (GREEN). Wire the ±60s supplement per 2.4 (Option A) and add its skew-tolerance test.

**Phase 5 — hardening pass.** Re-read `validate_response` against R2: assert the code path reads identity
**only** from `auth.get_nameid()/get_attributes()` (never `lxml`/regex over `saml_response_b64`). Confirm
`wantAssertionsSigned=True` and that the unsigned + XSW tests are genuinely RED without it (flip the flag
locally, watch them fail, flip back) — this proves the tests bite. Scoped green + a security-review-style
skim before handing to the routes aspect.

- **Runner:** `cd services/backend-api && pytest tests/test_saml_replay.py tests/test_saml_provider.py -v`

  tested (R2); replay store exactly-once under (simulated/real) concurrency.

## 6. Edge cases & failure modes

- **Null / missing NameID** → `SamlValidationError("assertion")` before any identity query (account-takeover
  guard kept from OIDC).
- **Assertion signed but Response not** → accepted (assertion sig is the trust anchor; `wantMessagesSigned
  =False`). **Response signed but Assertion not** → rejected (`wantAssertionsSigned=True`).
- **XSW (wrapped forged assertion)** → the library validates the signed assertion only; the forged
  subject must never appear in `ValidatedAssertion`. Explicit test.
- **Clock skew** → ±60s per 2.4; fixtures beyond tolerance rejected `expired`, within tolerance accepted.
- **Replay / duplicate ACS POST** → conditional UPDATE matches 0 rows on the second call → `REPLAY`.
- **Unsolicited (IdP-initiated or forged) response** → no matching request-id row → `UNSOLICITED`
  (SP-initiated only in slice 1).
- **Expired request row** (user dawdled past TTL) → `EXPIRED`.
- **Cleanup failure** must not fail a login — best-effort, logged, swallowed.
- **SSRF on IdP SSO URL** (private IP / `http://`) → `SsrfError` from `build_authn_request` (route maps to
  a generic `config` error next aspect).
- **`get_last_error_reason()` phrasing drift** across `python3-saml` versions → the 2.2 substring map may
  need a tweak; keep the default `assertion` so unknown reasons never fail open to success.
