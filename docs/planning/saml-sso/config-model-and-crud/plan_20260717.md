## 0. Key decisions locked for this aspect

1. **Cert storage & return shape — DECIDED: store full PEM, return a fingerprint, never the PEM.**
   `idp_x509_cert` is a **public** signing certificate (not a secret) so it is *not* Fernet-encrypted —
   it is stored as plaintext `Text`. But `SamlConfigResponse` returns a **SHA-256 fingerprint**
   (`cert_fingerprint`, colon-hex form, e.g. `AB:CD:…`) rather than echoing the whole PEM back. Rationale:
   (a) tidiness — mirrors OIDC's "return a hint, not the material" ergonomics (`secret_hint`); (b) the
   fingerprint is exactly what an admin cross-checks against their IdP console; (c) keeps the response
   small and stable. The `configured` boolean tells the UI a cert exists. This is the spec's open
   question ("fingerprint is tidier") resolved in favor of the fingerprint.
2. **PEM validation on save.** Parse `idp_x509_cert` with
   `cryptography.x509.load_pem_x509_certificate(...)` → any failure (garbage, truncated, wrong PEM
   type) raises **422**. `cryptography>=43` is already in `requirements.txt` (line 33) — no new dep.
3. **SSRF validation on save.** `idp_sso_url` must be `https://` + host-gated via
   `assert_host_not_ssrf` (`src/utils/ssrf.py`). Failure → **422**. (Mirrors the OIDC login-flow
   discipline; applied at *save* time here, and again at *use* time in the later login aspect.)
4. **Cross-provider single-SSO guard — one shared helper, two callers.** New module
   `src/api/routes/_sso_guard.py` exposing `assert_no_other_provider_enabled(db, *, enabling)`. The SAML
   route calls it with `enabling="saml"` (→ 422 if any OIDC config is enabled). The shipped OIDC route
   gains **exactly two lines** (one import + one call with `enabling="oidc"`) — its existing
   `_assert_no_other_enabled` (same-provider D5) stays **byte-identical**. See §6 (human-review checkpoint).
5. **No partial-unique index on `enabled`.** SQLite (test DB) does not enforce Postgres partial-unique
   indexes; the single-enabled invariant lives entirely in the route layer. Same as OIDC (verbatim from
   `c9d0e1f2a3b4`'s header note).
6. **Migration head — CONFIRMED sole leaf `n8o9p0q1r2s3`.** Verified live in this worktree: no
   revision has `down_revision = "n8o9p0q1r2s3"`, so it is the unique head. New migration(s) chain from
   it. Re-run `alembic heads` immediately before authoring (see §2 setup) — do not trust this static
   claim at build time.

### 4.1 `src/models/saml_config.py` (mirror `oidc_config.py`)
```python
"""
SAML SSO configuration model.

One row per organization. idp_x509_cert is the IdP's PUBLIC signing
certificate (PEM) — it is NOT a secret and NOT encrypted (contrast
oidc_config.client_secret, which is Fernet-encrypted). PEM validation and
the returned SHA-256 fingerprint are route-layer concerns; see
src/api/routes/saml_config.py.
"""
import sqlalchemy as sa
from sqlalchemy import (Column, Integer, String, Text, Boolean, DateTime,
                        ForeignKey, Index, UniqueConstraint, JSON)
from datetime import datetime
from .base import Base


class SamlConfig(Base):
    """Org-wide SAML 2.0 IdP connection (entity id, SSO URL, signing cert, domain allowlist). One row per org."""
    __tablename__ = "saml_configs"

    id = Column(Integer, primary_key=True, index=True)
    organization_id = Column(Integer, ForeignKey("organizations.id", ondelete="CASCADE"), nullable=False)
    idp_entity_id = Column(String(255), nullable=False)
    idp_sso_url = Column(String(512), nullable=False)          # HTTP-Redirect SSO URL; SSRF-gated on save
    idp_x509_cert = Column(Text, nullable=False)               # PUBLIC PEM signing cert — NOT encrypted
    email_attribute = Column(String(255), nullable=True)       # override for the email attr/NameID mapping
    enabled = Column(Boolean, nullable=False, default=False, server_default=sa.false())
    allowed_email_domains = Column(JSON, nullable=True)        # empty/absent = deny-all (same as OIDC)
    button_label = Column(String(255), nullable=False, default="Sign in with SSO", server_default="Sign in with SSO")

    created_at = Column(DateTime, default=datetime.utcnow, nullable=False)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow, nullable=False)

    __table_args__ = (
        UniqueConstraint('organization_id', name='uq_saml_configs_org_id'),
        Index('ix_saml_configs_org_id', 'organization_id'),
    )

    def __repr__(self):
        return f"<SamlConfig(id={self.id}, org={self.organization_id}, enabled={self.enabled})>"
```

### 4.4 `src/api/routes/_sso_guard.py` (new shared helper)
```python
"""
Cross-provider single-SSO guard (saml-sso M6).

At most ONE SSO protocol may be enabled per deployment. Enabling SAML must
fail if any OIDC config is enabled, and vice-versa. This is the CROSS-provider
half only; each provider's SAME-provider single-enabled D5 check stays in its
own route (oidc_config._assert_no_other_enabled / saml_config._assert_no_other_enabled)
so the shipped OIDC route's existing behavior + error string stay byte-stable.
"""
from fastapi import HTTPException, status
from sqlalchemy.orm import Session

from src.models.oidc_config import OidcConfig
from src.models.saml_config import SamlConfig

_LABELS = {"oidc": "OIDC", "saml": "SAML"}


def assert_no_other_provider_enabled(db: Session, *, enabling: str) -> None:
    """Raise 422 if the OTHER SSO provider has any enabled config anywhere in
    the deployment. `enabling` is 'oidc' or 'saml'."""
    other_model = SamlConfig if enabling == "oidc" else OidcConfig
    other_label = _LABELS["saml" if enabling == "oidc" else "oidc"]
    row = db.query(other_model).filter(other_model.enabled.is_(True)).first()
    if row is not None:
        raise HTTPException(
            status_code=status.HTTP_422_UNPROCESSABLE_ENTITY,
            detail=(f"A {other_label} SSO config is already enabled; "
                    f"only one SSO protocol may be active per deployment."),
        )
```

### 4.5 `src/api/routes/saml_config.py` (mirror `routes/oidc_config.py`)
Structure mirrors OIDC exactly. Key differences from the OIDC route:
- **No Fernet.** No `encrypt_api_key` / `LLM_ENCRYPTION_KEY` handling. Cert stored plaintext.
- **PEM validation helper** `_validate_pem_cert(pem: str) -> str` → parse via
  `x509.load_pem_x509_certificate(pem.encode())`; on `ValueError`/`Exception` raise 422
  (`"idp_x509_cert is not a valid PEM X.509 certificate."`). Returns the PEM unchanged.
- **Fingerprint helper** `_cert_fingerprint(pem: str) -> str` → load cert,
  `cert.fingerprint(hashes.SHA256())`, hex-encode, uppercase, colon-group.
- **SSRF+scheme helper** `_validate_idp_sso_url(url: str)` → require `https://`; extract host via
  `urllib.parse.urlparse`; `assert_host_not_ssrf(host)`; map `SsrfError`/scheme failure → 422.
- Reuse `_normalize_domains` (copy verbatim — deny-all semantics identical).
- `_assert_no_other_enabled(db, org_id)` — copy the OIDC same-provider D5 helper verbatim, swapping
  `OidcConfig`→`SamlConfig` and the message to
  `"Another SAML config is already enabled; only one may be active per deployment."`.

Schemas:
```python
class SamlConfigUpdateRequest(BaseModel):
    idp_entity_id: str = Field(..., min_length=1)
    idp_sso_url: str = Field(..., min_length=1)
    idp_x509_cert: Optional[str] = Field(None, min_length=1)   # optional on update (preserve if omitted)
    email_attribute: Optional[str] = None
    enabled: bool = False
    allowed_email_domains: List[str] = Field(default_factory=list)
    button_label: Optional[str] = None

class SamlConfigResponse(BaseModel):
    configured: bool
    idp_entity_id: Optional[str] = None
    idp_sso_url: Optional[str] = None
    cert_fingerprint: Optional[str] = None        # SHA-256, colon-hex — NEVER the raw PEM
    email_attribute: Optional[str] = None
    enabled: bool = False
    allowed_email_domains: List[str] = Field(default_factory=list)
    button_label: Optional[str] = None
```

PUT handler order of operations (mirror OIDC's, swap secret→cert):
1. `domains = _normalize_domains(payload.allowed_email_domains)`
2. `_validate_idp_sso_url(payload.idp_sso_url)` (always — even when disabled).
3. If `payload.enabled`: `_assert_no_other_enabled(db, org.id)` **and**
   `assert_no_other_provider_enabled(db, enabling="saml")`.
4. Cert resolution: if `payload.idp_x509_cert` provided → `_validate_pem_cert(...)`, use it; elif existing
   → keep `existing.idp_x509_cert`; else → 422 (`"idp_x509_cert is required when creating a new SAML config."`).
5. `button_label = payload.button_label or (existing.button_label if existing else None) or "Sign in with SSO"`.
6. Upsert row; `db.commit()`; `db.refresh(row)`; return `_build_response(row)` (computes fingerprint).

`_build_response(row)` sets `configured=True`, echoes the non-secret fields, and sets
`cert_fingerprint=_cert_fingerprint(row.idp_x509_cert)`.

GET / DELETE: identical shape to OIDC (GET → `configured:false` when absent; DELETE → 404 when absent,
else `{"success": True, "message": "SAML config deleted."}`). All three verbs
`dependencies=[Depends(require_admin_or_owner)]`, scoped by `get_current_org`.

## 8. Edge cases & risks

- **Multi-tenancy / org-scoping.** Every CRUD query filters `SamlConfig.organization_id == current_org.id`
  (from `get_current_org`, derived from the JWT). The same-provider D5 helper filters
  `organization_id != org_id` so an org may re-enable/edit its own row. The status probe is deployment-wide
  (`enabled.is_(True)` across all orgs) — intentional and identical to OIDC (single-tenant self-host
  reality). Add a test that org B cannot see org A's config via GET (returns `configured:false` for B).
- **Cert return shape (DECIDED §0.1): fingerprint, not PEM.** If a future aspect needs the raw PEM in the
  UI (e.g. to display for copy), that is an additive response field then — not now.
- **PEM edge cases:** DER (not PEM), cert chains (multiple certs concatenated — `load_pem_x509_certificate`
  reads only the first; acceptable, IdP signing certs are single), expired certs (we do NOT reject on
  validity dates at *config* time — an IdP may rotate; validity is an assertion-time concern in the later
  aspect). Only reject un-parseable PEM here.
- **`idp_sso_url` at 512 chars** — longer than OIDC's 255 issuer, because IdP SSO endpoints often carry
  path/query. Enforced by column length; Pydantic has no max but DB truncation on Postgres would error —
  optionally add `max_length=512` to the Pydantic field for a friendly 422 (nice-to-have).
- **SSRF on save vs. use:** validated here on save; the later login aspect must re-validate at use time
  (TOCTOU / DNS-rebind window is documented in `ssrf.py`). Not this aspect's job to close.
- **`auth_provider` has no DB constraint** — adding `"saml"` is a comment-only change; no migration. A
  user linked via both Google/OIDC/SAML uses `"both"`-style values (login aspect's concern).
- **No `LLM_ENCRYPTION_KEY` dependency** — a deployment can configure SAML with the key entirely unset
  (unlike OIDC). Confirm no code path in the SAML route imports/΄requires encryption.
- **Migration head drift (R6 from PRD):** if `alembic heads` in §2 shows anything other than
  `n8o9p0q1r2s3`, STOP — a concurrent branch may have added a revision; rebase/reconcile the
  `down_revision` before authoring.
