1. Add to `workflow_service.py`:
   ```python
   def apply_status_change(db, feedbacks, new_status, *, organization_id,
                           actor_id=None, actor_label, resolution_note=None):
       """Mutate status + emit timeline events for the items whose status actually
       changes. Returns list[(feedback, old_status)]. Caller validates new_status,
       commits, and handles cache/webhook/WS. actor_id may be None (API-key writes)."""
       changed = []
       for fb in feedbacks:
           old = fb.workflow_status
           if old == new_status:
               continue
           fb.workflow_status = new_status
           meta = {"resolution_note": resolution_note} if (resolution_note and new_status == "resolved") else None
           create_workflow_event(db, fb.id, organization_id, actor_id,
                                 "status_changed", old_value=old, new_value=new_status, metadata=meta)
           changed.append((fb, old))
       return changed
   ```
   Also add `dispatch_status_webhooks(db, org_id, changed, new_status, changed_by)` OR keep the webhook loop in the route — **decision:** extract a `dispatch_status_webhooks(db, org_id, changed, new_status, changed_by_label)` helper too so the public route reuses it. Keep `cache_invalidate` + WS `emit_event` in each route (they're already thin).
2. Refactor `change_status` to call `apply_status_change(..., actor_id=current_user.id, actor_label=current_user.email, ...)` then `db.commit()`, cache invalidate, `dispatch_status_changed`, `dispatch_status_webhooks(..., changed_by_label=current_user.email)`, `emit_event`. **Behavior identical.**

**Files:**

- `services/backend-api/tests/test_public_api_write.py` — **new**.

**Request model (wire format — OQ2 decision: nested correction):**
```python
class PublicCorrectionInput(BaseModel):
    field: Literal["pain_point", "feature_request", "sentiment"]
    corrected_value: str = Field(..., min_length=1)

class PublicFeedbackUpdate(BaseModel):
    workflow_status: Optional[Literal["new","in_review","resolved","closed"]] = None
    resolution_note: Optional[str] = None
    correction: Optional[PublicCorrectionInput] = None
```
Map `field` → (`correction_type`, current stored value read for `original_value`): `pain_point`→`("category", fb.pain_point_category)`, `feature_request`→`("category", fb.feature_request_category)`, `sentiment`→`("sentiment", fb.sentiment_label)`.

**Route behavior:**
```python
@router.patch("/feedback/{feedback_id}", response_model=PublicFeedbackItem,
              dependencies=[Depends(require_scope("write"))],
              summary="Update a feedback item (public API)")
async def public_update_feedback(feedback_id, data: PublicFeedbackUpdate,
                                 auth: ApiKeyAuth = Depends(verify_api_key),
                                 db: Session = Depends(get_db)):
    if data.workflow_status is None and data.correction is None:
        raise HTTPException(400, "Nothing to update")
    fb = db.query(FeedbackItem).filter(FeedbackItem.id == feedback_id,
             FeedbackItem.organization_id == auth.organization_id).first()
    if fb is None: raise HTTPException(404, "Feedback not found")
    key_label = f"api-key:{auth.key_row.name}"
    if data.workflow_status is not None:
        changed = apply_status_change(db, [fb], data.workflow_status,
                     organization_id=auth.organization_id, actor_id=None,
                     actor_label=key_label, resolution_note=data.resolution_note)
        db.commit()
        if changed:
            cache_invalidate(...dashboard/analytics...)
            dispatch_status_webhooks(db, auth.organization_id, changed, data.workflow_status, key_label)
            await emit_event(org_id=auth.organization_id, event_type="workflow:status_changed",
                             data={"feedback_ids":[fb.id],"new_status":data.workflow_status})
    if data.correction is not None:
        ctype, original = _resolve_correction(fb, data.correction.field)
        create_ai_correction(db, organization_id=auth.organization_id, user_id=None,
            correction_type=ctype, entity_type="feedback_item", entity_id=fb.id,
            signal="correction", original_value=original,
            corrected_value=data.correction.corrected_value, feedback_text=fb.text)
    db.refresh(fb)
    return PublicFeedbackItem.model_validate(fb)
```
> Note: `create_ai_correction` commits internally; call it after the status commit. `user_id=None` is allowed (see edge cases). Skip `dispatch_status_changed` (in-app user notifications) for API-key writes — it's user-facing; the webhook + timeline event still fire.

## Edge cases & error handling
- `actor_id=None` on the timeline event — allowed (nullable FK). Verify `TimelineEventResponse` serialization elsewhere tolerates null `actor_id`/`actor_email` (internal timeline GET) — if it asserts non-null, that's a pre-existing read-path concern; note but don't fix here unless a test breaks.
- `user_id=None` on `AICorrection` — **verified nullable** (`ondelete="SET NULL", nullable=True`). No migration needed; the "no migration" assumption holds.
- Webhook/notification dispatch stays best-effort (try/except) — a failing webhook must not fail the PATCH.
- Multi-tenancy: every query filtered by `auth.organization_id`; cross-org → 404 not 403.
- `resolution_note` only attaches to metadata when `new_status == "resolved"` (mirror internal).
