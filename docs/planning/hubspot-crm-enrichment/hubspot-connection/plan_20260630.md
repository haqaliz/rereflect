### Critical risk flagged in PRD (R6)

`services/backend-api/src/utils/encryption.py:14` — `_get_fernet()` raises
`ValueError("LLM_ENCRYPTION_KEY environment variable is not set")` when the env
var is absent. This will propagate uncaught through `encrypt_api_key` (line 19),
producing an unhandled 500 on `POST /connect`. The route **must** catch
`ValueError` from `encrypt_api_key` and return HTTP 422 with an actionable
message. A test must cover this path before any production code is written
(RED phase requirement).

### Do NOT repeat the Linear plaintext mistake

`services/backend-api/src/models/linear_integration.py:12` stores `access_token`
as plaintext despite the inline comment saying "Fernet-encrypted". The correct
pattern is in `services/backend-api/src/api/routes/webhooks.py:411-412` where
`encrypt_api_key` is called before persisting and `decrypt_api_key` is called on
every read. HubSpot must follow the webhooks.py pattern, not the linear pattern.

   ```python
   HUBSPOT_ACCOUNT_INFO_URL = "https://api.hubapi.com/account-info/v3/details"

   def _validate_hubspot_token(access_token: str) -> dict:
       """
       Validate a HubSpot private-app token by calling /account-info/v3/details.
       Returns portal metadata dict on success.
       Raises HTTPException 422 on auth failure.
       Raises HTTPException 502 on network error.
       """
       try:
           with httpx.Client(timeout=10.0) as http_client:
               resp = http_client.get(
                   HUBSPOT_ACCOUNT_INFO_URL,
                   headers={"Authorization": f"Bearer {access_token}",
                            "Content-Type": "application/json"},
               )
               resp.raise_for_status()
               return resp.json()
       except httpx.HTTPStatusError as exc:
           if exc.response.status_code in (401, 403):
               raise HTTPException(
                   status_code=status.HTTP_422_UNPROCESSABLE_ENTITY,
                   detail="HubSpot token is invalid or lacks required permissions.",
               )
           raise HTTPException(
               status_code=status.HTTP_502_BAD_GATEWAY,
               detail=f"HubSpot API returned {exc.response.status_code}.",
           )
       except httpx.RequestError as exc:
           raise HTTPException(
               status_code=status.HTTP_502_BAD_GATEWAY,
               detail=f"Could not reach HubSpot API: {exc}",
           )
   ```

   **POST /connect — R6 safeguard** (critical):

   ```python
   @router.post("/connect", response_model=HubSpotConnectResponse,
                dependencies=[Depends(require_admin_or_owner),
                               Depends(require_feature("hubspot_integration"))])
   def hubspot_connect(
       payload: HubSpotConnectRequest,
       current_org: Organization = Depends(get_current_org),
       current_user: User = Depends(get_current_user),
       db: Session = Depends(get_db),
   ):
       # 1. Validate token against HubSpot before encrypting
       portal_meta = _validate_hubspot_token(payload.access_token)

       # 2. Encrypt — catch ValueError from missing LLM_ENCRYPTION_KEY (R6)
       try:
           encrypted = encrypt_api_key(payload.access_token)
       except ValueError as exc:
           raise HTTPException(
               status_code=status.HTTP_422_UNPROCESSABLE_ENTITY,
               detail=(
                   "Cannot store HubSpot token: LLM_ENCRYPTION_KEY is not set. "
                   "Set this environment variable and restart the service to connect "
                   "a CRM integration."
               ),
           ) from exc

       hint = get_key_hint(payload.access_token)  # "...last4"

       # 3. Upsert
       existing = (
           db.query(HubSpotIntegration)
           .filter(HubSpotIntegration.organization_id == current_org.id)
           .first()
       )
       if existing:
           existing.access_token = encrypted
           existing.token_hint = hint
           existing.hub_id = str(portal_meta.get("portalId", ""))
           existing.portal_name = portal_meta.get("companyName") or portal_meta.get("timeZone")
           existing.arr_property_name = payload.arr_property_name
           existing.connected_by_user_id = current_user.id
           existing.connected_at = datetime.utcnow()
           existing.is_active = True
           existing.updated_at = datetime.utcnow()
           integration = existing
       else:
           integration = HubSpotIntegration(
               organization_id=current_org.id,
               access_token=encrypted,
               token_hint=hint,
               hub_id=str(portal_meta.get("portalId", "")),
               portal_name=portal_meta.get("companyName") or portal_meta.get("timeZone"),
               arr_property_name=payload.arr_property_name,
               connected_by_user_id=current_user.id,
               connected_at=datetime.utcnow(),
               is_active=True,
           )
           db.add(integration)
       db.commit()
       db.refresh(integration)

       logger.info("HubSpot connected for org %s (portal %s)",
                   current_org.id, integration.hub_id)
       return HubSpotConnectResponse(
           connected=True,
           portal_name=integration.portal_name,
           hub_id=integration.hub_id,
           token_hint=integration.token_hint,
       )
   ```

## 4. Testing Strategy

### Strict TDD discipline

Every file listed above is written **test-first**:
1. Write the failing test (RED). Run it. Confirm it fails for the expected
   reason (import error → file-not-found, assertion error → logic).
2. Write the minimum production code to make it pass (GREEN).
3. Refactor without breaking tests.

### Backend — pytest with in-memory SQLite

Pattern established in `services/backend-api/tests/conftest.py`:

- `db` fixture (`scope="function"`) creates the full schema via
  `Base.metadata.create_all(bind=engine)` on each test, drops after.

## 6. Edge Cases and Error Handling

### R6 — Missing LLM_ENCRYPTION_KEY (critical)

`encrypt_api_key` (line 19 of `services/backend-api/src/utils/encryption.py`)
raises `ValueError("LLM_ENCRYPTION_KEY environment variable is not set")` when
called with no key configured. `POST /connect` must wrap this call in
`try/except ValueError` and return HTTP 422 with an operator-actionable message.
This is a hard requirement: a test covering this path must exist in RED phase
and pass in GREEN phase before any deploy.

### Token validation failure (HubSpot returns 401/403)

`_validate_hubspot_token` translates `httpx.HTTPStatusError` with status 401
or 403 to HTTP 422 ("HubSpot token is invalid or lacks required permissions").
Other 4xx/5xx codes map to HTTP 502 ("HubSpot API returned ..."). This prevents
storing an invalid token silently.

### Network error reaching HubSpot

`httpx.RequestError` (DNS, connection refused, timeout) maps to HTTP 502.
This is important for self-hosted environments where HubSpot may be unreachable
due to network configuration; the error is surfaced to the operator, not swallowed.

### Duplicate connect (upsert semantics)

If an org calls `POST /connect` twice, the second call updates the existing row
(re-encrypts the new token, resets metadata). This is intentional: the operator
may rotate their HubSpot private-app token. A test covers this upsert path.

### Organization scoping

Every query on `HubSpotIntegration` filters by `organization_id = current_org.id`.
`current_org` is derived from the authenticated user's JWT claim via
`get_current_org` → `current_user.organization` (see
`services/backend-api/src/api/dependencies.py:99-101`). This means a valid
JWT from org A cannot ever reach org B's HubSpot row — the org boundary is
enforced at the dependency layer before any route handler logic runs.

### Member role (RBAC)

`require_admin_or_owner` (line 255 of `services/backend-api/src/api/dependencies.py`)
raises HTTP 403 when `current_user.role == 'member'`. All four endpoints
carry `dependencies=[Depends(require_admin_or_owner), ...]`. The frontend
detail page additionally redirects members to `/settings/preferences` before
rendering the form.

### Disconnect with no connection

`DELETE /disconnect` returns HTTP 404 if no row exists for the org. This is
consistent with the `_require_active_integration` helper pattern in
`services/backend-api/src/api/routes/linear_integration.py:181-189`.

### POST /test with corrupted token

If `access_token` in the DB is corrupted (e.g. key rotation without
re-encrypting), `decrypt_api_key` raises `cryptography.fernet.InvalidToken`.
`POST /test` catches all exceptions and returns `{"success": false, "message": "..."}`,
never a 500. The test coverage for this uses a deliberately malformed ciphertext
in the DB row.

### arr_property_name defaults

`arr_property_name` defaults to `"annualrevenue"` (the default HubSpot
company property for Annual Revenue). The operator may provide a custom name
at connect time (e.g. `"annual_recurring_revenue"`). The value is stored on
the integration row and later used by the hubspot-sync aspect when reading
company data. If not provided, the default applies both at the Pydantic model
level and at the database column server_default.

### SQLite compatibility note for tests

SQLite (in-memory test DB) does not enforce `UniqueConstraint` the same way
PostgreSQL does in all edge cases. The `test_organization_id_unique_constraint`
test must trigger a commit after the second insert to force constraint checking.
Use `db.commit()` inside a `pytest.raises(Exception)` block — the exact
exception type varies between SQLite and PostgreSQL, so `Exception` is the
correct base class to catch in tests.
