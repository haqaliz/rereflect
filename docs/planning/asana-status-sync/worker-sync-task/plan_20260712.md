## Structural differences from `jira_sync.py` (read before mirroring)

The Jira task is the template, but Asana's link model is shaped differently. Do not blindly copy — apply these deltas:

| Concern | Jira | Asana |
|---|---|---|
| Batch fetch | `client.search_issues(chunk<=50)` — one call, many keys | **No batch endpoint.** One `client.get_task(gid)` **per link** (PRD risk: N-GET fan-out). No chunking loop. |
| Link status column | `jira_status` (name String) + `jira_status_category` | **No name column.** `asana_completed` (Boolean) + `asana_status_category` (String). |
| Category source | `info["category"]` from Jira's statusCategory | `asana_category(fetched.completed)` → `"done"` / `"new"` |
| `decide_link_update` name arg | Jira status name | a synthetic label (`"Completed"` / `"Open"`) — used only for logging/metadata, **not persisted** (no name column) |
| Missing task | key absent from batch dict → skip link | `get_task` raises `AsanaNotFoundError` (404) OR returns None → **catch/skip that one link, continue the loop** |
| Event metadata | `{source, jira_status, jira_status_category, jira_issue_key}` | `{"source":"asana","asana_task_gid":…,"asana_completed":…}` |

Everything else — fan-out filter, retryable wrapper, `_persist_terminal_status` fresh-session write, missing-key
no-retry, auth-error record-without-disconnect, seed/noop/changed handling, `most_advanced` +
`resolve_target_status`, no-op-on-equal-target — is mirrored **exactly**.

## Implementation Phases (TDD-first)

### Phase 1 — Lift `apply_status_change_worker` to a shared module (characterization-locked)

**Goal:** one provider-agnostic status writer, imported by both Jira and Asana tasks, with **zero behavior
change to Jira**.

**Decision (recommended): LIFT, do not copy.** `apply_status_change_worker` in `jira_sync.py` (lines ~130-179)
is already provider-neutral: it takes `actor_label` and a free-form `metadata` dict and hardcodes nothing
Jira-specific. Lifting it to a shared module and importing from both tasks is the low-drift choice.

**Tradeoff (state it in the PR):**
- *Lift* — single source of truth; the "one event, no-op-on-equal, `actor_id=None`, no outbound webhook"
  invariants can never diverge between providers. **Cost:** touches `jira_sync.py` (a shipped, merged file),
  so it must be proven byte-identical. Mitigation = the characterization lock below.
- *Copy* — zero risk to the Jira file. **Cost:** two copies of identical logic that will drift the first time
  someone changes one (e.g. adds outbound webhook dispatch to one provider and forgets the other).

Given both providers want *identical* semantics and this is exactly the reuse the PRD's "provider-agnostic core"
investment was for, **lift wins**. The characterization lock makes the cost safe.

**Files:**
- `services/worker-service/src/services/status_writer.py` (new)
- `services/worker-service/src/tasks/jira_sync.py` (modify — delete local def, import from shared)

**Steps:**
1. **GREEN baseline (characterization):** run `pytest tests/test_jira_sync_task.py -v` and confirm it passes
   *before touching anything*. That suite asserts the exact event shape (`old_value`, `new_value`,
   `metadata_["source"]=="jira"`, `metadata_["jira_issue_key"]`, exactly one event, no-op cases). It **is** the
   characterization test that locks the writer's behavior — no new characterization test file is needed.
2. Create `src/services/status_writer.py` containing `apply_status_change_worker(db, feedback, new_status, *,
   organization_id, actor_label, metadata=None) -> bool` — moved **verbatim** from `jira_sync.py` (same
   no-op-on-equal guard, same `actor_id=None`, same single `FeedbackWorkflowEvent(... metadata_=metadata,
   created_at=datetime.utcnow())`, same "no outbound webhook — DEFERRED" docstring note, generalized so the log
   line is provider-neutral, e.g. `"status_writer: %s changed feedback_id=..."`). Import `FeedbackWorkflowEvent`
   lazily inside the function (as today) to keep import order safe.
3. In `jira_sync.py`, delete the local `def apply_status_change_worker` and add
   `from src.services.status_writer import apply_status_change_worker`. Leave the call site untouched.
4. **RE-RUN the lock:** `pytest tests/test_jira_sync_task.py -v` must be **still fully green, same counts**.
   That proves the lift is byte-identical. If a single assertion moves, revert step 2's generalization and keep
   the log text identical too.

**Validation:** `cd services/worker-service && source venv/bin/activate && pytest tests/test_jira_sync_task.py -v`
(all pass, unchanged). No Asana code yet.

**Commit:** `refactor(worker): lift apply_status_change_worker into shared status_writer (jira byte-identical)`

### Phase 3 — GREEN: implement `asana_sync.py`

**Goal:** make Phase 2 pass by cloning `jira_sync.py` with the structural deltas above.

**File:** `services/worker-service/src/tasks/asana_sync.py` (new)

**Module skeleton (mirror `jira_sync.py`):**
```python
from __future__ import annotations
import logging, os
from datetime import datetime
from typing import Any, Dict, Optional
from celery import shared_task

from src.clients.asana import (
    AsanaAuthError, AsanaClient, AsanaTransientError, AsanaNotFoundError,
    asana_category,                       # completed(bool) -> "done" | "new"
)
from src.database import get_db_session
from src.services.status_sync_core import decide_link_update, most_advanced, resolve_target_status
from src.services.status_writer import apply_status_change_worker   # lifted in Phase 1

logger = logging.getLogger(__name__)
```
*(Import names for the client/adapter follow the asana-client-get-task aspect's contract — reconcile exact
symbol names when that aspect lands; adjust if e.g. the adapter lives elsewhere or 404 is signalled by
`get_task` returning `None` rather than raising `AsanaNotFoundError`.)*

**Steps:**
1. `_decrypt(token)` — identical to Jira's (local Fernet, `LLM_ENCRYPTION_KEY`, missing → `ValueError`).
2. `_persist_terminal_status(integration_id, status, error)` — identical to Jira's, but query the worker
   `AsanaIntegration` mirror in a **fresh** `get_db_session()` (the same rollback rationale as Jira/Zendesk).
3. **`_sync_asana_org_body(integration_id, db, client=None) -> Dict[str, Any]`:**
   - Load `AsanaIntegration` by id; guard `not integ → not_found`, `not is_active → inactive`,
     `not status_sync_enabled → disabled`.
   - `owns_client = client is None`. If owns_client: `try: token = _decrypt(integ.api_token)` /
     `except ValueError: integ.last_sync_status="error"; integ.last_error="missing_encryption_key"; db.flush();
     return {"status":"error","reason":"missing_encryption_key"}` (no retry). Then
     `client = AsanaClient(token)` (Asana uses a Bearer PAT — no site_url/email).
   - `try:` block:
     - `links = db.query(FeedbackAsanaTask).filter(FeedbackAsanaTask.organization_id == integ.organization_id).all()`
     - `seeded = 0; changed_links = 0; changed_feedback_ids = set()`
     - **Per-link loop (one GET each — no batch):**
       ```python
       for link in links:
           try:
               task = client.get_task(link.asana_task_gid)   # {"completed":..., ...}
           except AsanaNotFoundError:
               continue                                       # 404 → leave link unchanged
           if task is None:
               continue
           completed = bool(task.get("completed"))
           category = asana_category(completed)               # "done" | "new"
           label = "Completed" if completed else "Open"       # synthetic name (not persisted)
           action, _name, category = decide_link_update(category, label, link.asana_status_category)
           if action == "noop":
               continue
           link.asana_completed = completed
           link.asana_status_category = category
           link.last_status_synced_at = datetime.utcnow()
           if action == "seed":
               seeded += 1
           else:  # "changed"
               changed_links += 1
               changed_feedback_ids.add(link.feedback_id)
       ```
       **Note:** `AsanaTransientError` / `AsanaAuthError` are NOT caught here — they propagate to the outer
       `try` (auth) / to the caller (transient). Only `AsanaNotFoundError` is swallowed per-link.
     - **Per-changed-feedback resolution (mirror Jira exactly):**
       ```python
       status_changes = 0
       for feedback_id in changed_feedback_ids:
           feedback = db.query(FeedbackItem).filter(FeedbackItem.id == feedback_id).first()
           if feedback is None:
               continue
           live_links = [l for l in links if l.feedback_id == feedback_id]
           categories = [l.asana_status_category for l in live_links if l.asana_status_category]
           top_category = most_advanced(categories)
           if top_category is None:
               continue
           target = resolve_target_status(top_category, integ.status_mapping)
           if target is None or target == feedback.workflow_status:
               continue
           driving_link = next(
               (l for l in live_links if l.asana_status_category == top_category), live_links[0]
           )
           applied = apply_status_change_worker(
               db, feedback, target,
               organization_id=integ.organization_id,
               actor_label="asana-sync",
               metadata={
                   "source": "asana",
                   "asana_task_gid": driving_link.asana_task_gid,
                   "asana_completed": driving_link.asana_completed,
               },
           )
           if applied:
               status_changes += 1
       ```
     - `integ.last_synced_at = datetime.utcnow(); integ.last_sync_status = "success"; integ.last_error = None;
       db.flush()`; return `{"status":"success","links_seen":len(links),"seeded":seeded,
       "changed_links":changed_links,"status_changes":status_changes}`.
   - `except AsanaAuthError as exc:` — record `last_sync_status="error"`, `last_error=str(exc)[:500]`,
     `db.flush()`, `return {"status":"error","reason":"auth_error"}` (no raise, no disconnect — D7 parity).
     `AsanaTransientError` is deliberately **not** caught → propagates to `sync_asana_org`.
   - `finally:` — `if owns_client: client.close()`.
4. **`sync_all_asana()`** — `@shared_task(name="src.tasks.asana_sync.sync_all_asana")`; open `get_db_session()`,
   query `AsanaIntegration` where `is_active.is_(True) AND status_sync_enabled.is_(True)`;
   `return {"status":"no_integrations","queued":0}` if none; else per-org `try: sync_asana_org.delay(integ.id);
   queued += 1` / `except Exception: log org_id + integration_id` (one failure never aborts the batch);
   `return {"status":"queued","queued":queued}`.
5. **`sync_asana_org(self, integration_id)`** — `@shared_task(bind=True, max_retries=3, default_retry_delay=30,
   name="src.tasks.asana_sync.sync_asana_org")`; `with get_db_session() as db: try: return
   _sync_asana_org_body(integration_id, db)` / `except AsanaTransientError as exc: log warning;
   _persist_terminal_status(integration_id, "retrying", str(exc)); raise self.retry(exc=exc)`. Keep the Jira
   comment about *not* `db.flush()`-ing before the retry (avoids a self-deadlock against the fresh session).
6. **R3 parity:** never log the PAT. Log lines use `integration_id` / `organization_id` only.

**Validation:** `cd services/worker-service && source venv/bin/activate && pytest tests/test_asana_sync_task.py -v`

## Edge Cases & Error Handling

- **⭐ Manual status untouched:** first observation of any link is **seed** (`stored_category IS NULL`) → link
  state only, never a feedback move. A status the poller never drove is never reverted. (Test 1.)
- **No-op on equal target:** `apply_status_change_worker` returns False and writes nothing when
  `target == feedback.workflow_status` — bounds the reopen tug. (Tests 4, 6.)
- **Reopen tug (accepted, audited):** a sync-driven `resolved` reverts to `new` on reopen, with one
  `source=asana` event so it's explainable. (Test 5.)
- **Multi-task feedback:** `most_advanced` prevents regression when one of several linked tasks reopens. (Test 7.)
- **404 on a task:** skip that one link, continue; org still reports `success`. (Test 8.)
- **429 / 5xx (`AsanaTransientError`):** propagate to `sync_asana_org` → `self.retry` (max 3, 30s). `Retry-After`
  honoring lives in the client aspect. (Tests 9, 15.)
- **401 / 403 (`AsanaAuthError`):** record `last_sync_status="error"` + `last_error`, keep `is_active=True`, no
  retry. (Test 10.)
- **Missing `LLM_ENCRYPTION_KEY`:** non-transient → `{"status":"error","reason":"missing_encryption_key"}`, no
  retry, no client construction. (Test 11.)
- **Disabled / inactive org:** zero `get_task` calls. (Tests 13, 14.)
- **Batch isolation:** one org raising in fan-out never aborts the others (per-org try/except).
- **Multi-tenancy:** every worker query filters `organization_id == integ.organization_id`.
- **Unknown category:** `resolve_target_status` returns None → skip (defensive; Asana emits only done/new in v1).
