## 4. Implementation phases (TDD)

### Phase 1 — Base interface + provider contract
- **Goal:** Define the abstraction every provider implements; no network.
- **Files:** `base.py`, `__init__.py`, `providers/__init__.py`.
- **Steps:**
  1. **RED:** `tests/embeddings/test_base.py` — assert `EmbeddingProvider` is abstract,
     `embed(text: str) -> list[float]` is required, and a `dimension` property exists.
  2. **GREEN:** `base.py`:
     - `class EmbeddingProvider(ABC)` with `@abstractmethod embed(self, text: str) -> list[float]`
       and `@property @abstractmethod dimension(self) -> int`.
     - Default model constants documented per provider.
  3. **REFACTOR:** docstrings, typing.
- **Validation:** `pytest tests/embeddings/test_base.py -v`.
- **Commit:** `feat(embeddings): EmbeddingProvider base interface`

### Phase 2 — OpenAI + OpenAI-compatible providers
- **Goal:** Parity with the current hardcoded call, plus the keyless local path.
- **Files:** `providers/openai.py`, `providers/openai_compatible.py`.
- **Steps:**
  1. **RED:** `tests/embeddings/test_openai_provider.py` — mock `openai.OpenAI`; assert
     `OpenAIEmbeddingProvider(api_key="k", model="text-embedding-3-small").embed("hi")`
     returns a 1536-len list and `.dimension == 1536`; assert empty api_key raises a clear error.
  2. **RED:** `tests/embeddings/test_openai_compatible_provider.py` — construct with
     `base_url="http://localhost:11434/v1"`, **no api_key**; mock the client to return a
     768-len vector; assert `embed()` returns it and `.dimension == 768` (dimension derived
     from the **actual response length**, not hardcoded); assert missing `base_url` raises.
  3. **GREEN:** implement both. OpenAI mirrors `template_matcher._call_embedding_api`
     (`client.embeddings.create(model=..., input=text)` → `response.data[0].embedding`).
     OpenAI-compatible uses `openai.OpenAI(base_url=base_url, api_key=api_key or "ollama")`
     (dummy key when keyless, matching the worker pattern) and the same `.embeddings.create`.
     `dimension` = `len` of the last/first returned vector (cache after first call or accept a
     configured hint; default to response length).
  4. **REFACTOR:** factor shared OpenAI-SDK call into a small helper if duplication is real.
- **Validation:** `pytest tests/embeddings/ -v`.
- **Commit:** `feat(embeddings): OpenAI + OpenAI-compatible embedding providers`

### Phase 3 — Google provider
- **Goal:** Gemini embeddings normalized to `list[float]`.
- **Files:** `providers/google.py`. **First:** verify the Google SDK is importable in
  backend-api (`python -c "import google.generativeai"`); if not, add the pinned package used
  by the worker's Google provider to `services/backend-api/requirements.txt` and note it in §6.
- **Steps:**
  1. **RED:** `tests/embeddings/test_google_provider.py` — mock the Google client; assert
     `embed()` returns a `list[float]` and `.dimension` matches the response (e.g. 768).
  2. **GREEN:** implement using `text-embedding-004` default; normalize the SDK's response
     shape to a flat `list[float]`.
  3. **REFACTOR.**
- **Validation:** `pytest tests/embeddings/ -v`.
- **Commit:** `feat(embeddings): Google (Gemini) embedding provider`

### Phase 4 — Factory
- **Goal:** Name → provider instance, mirroring `LLMProviderFactory.create`.
- **Files:** `factory.py`.
- **Steps:**
  1. **RED:** `tests/embeddings/test_factory.py` — `create("openai", api_key="k", model=None)`
     → `OpenAIEmbeddingProvider`; `create("openai_compatible", base_url=..., model="nomic-embed-text")`
     → compatible provider; `create("ollama", ...)` defaults base_url to localhost:11434/v1;
     `create("google", ...)` → Google; `create("anthropic", ...)` raises a clear
     "no first-party embeddings API" `ValueError`; unknown/empty raises `ValueError`.
  2. **GREEN:** implement `EmbeddingProviderFactory.create(provider, *, api_key=None,
     base_url=None, model=None)` with per-provider default models. Lazy imports per branch
     (mirror worker factory) so a missing optional SDK doesn't break the others.
  3. **REFACTOR.**
- **Validation:** `pytest tests/embeddings/ -v`.
- **Commit:** `feat(embeddings): EmbeddingProviderFactory`

### Phase 5 — Org resolver (degrade signal)
- **Goal:** Single entry the consumers call: org → a ready embedder, or `None` to degrade.
- **Files:** `resolver.py`.
- **Steps:**
  1. **RED:** `tests/embeddings/test_resolver.py` (uses a test DB session / mocks):
     - Org with `default_provider="openai"` + a valid OpenAI BYOK key → returns an embedder
       whose provider is openai; reads key via `resolve_org_byok_key`.
     - Org with `default_provider="openai_compatible"` + `base_url` set + **no key** →
       returns a keyless local embedder (no BYOK lookup required).
     - Org with `openai_compatible` but **no base_url** → returns `None` (cannot resolve).
     - Org with `openai` but **no BYOK key** → returns `None`.
     - No `OrgAIConfig` row → returns `None`.
  2. **GREEN:** implement `resolve_embedding_provider(org_id, db)`:
     - Load `OrgAIConfig`; pick `provider = config.default_provider`,
       `base_url = config.base_url`, `model = getattr(config, "model_embeddings", None)`
       (column added later in `template-matching-local` S1 — use `getattr` so this aspect
       doesn't depend on it).
     - For `_LOCAL_PROVIDERS = {"ollama", "openai_compatible"}`: require `base_url`
       (or Ollama default); **no** BYOK lookup; build via factory with `api_key=None`.
     - Else (cloud): `api_key = resolve_org_byok_key(provider, org_id, db)`; if `None`
       return `None`; build via factory.
     - Wrap construction in try/except → log + return `None` (never raise to caller).
     - Return a small dataclass `ResolvedEmbedder(provider: str, embedder, dimension_hint)`.
  3. **REFACTOR:** keep `_LOCAL_PROVIDERS` as a module frozenset mirroring the worker’s.
- **Validation:** `pytest tests/embeddings/ -v` (full package green).
- **Commit:** `feat(embeddings): org-scoped embedding provider resolver with degrade-to-None`

## 6. Environment & secrets
- No new required env. Local path is keyless (`OrgAIConfig.base_url`). Cloud uses existing
  BYOK (`OrgApiKey` + `LLM_ENCRYPTION_KEY`, already configured).
- If Phase 3 must add the Google SDK, pin it to the **same version** as
  `services/worker-service/requirements.txt` and record here: `google-generativeai==<match>`.

## 7. Edge cases & error handling
- Empty/whitespace text → return a zero-cost no-op or let provider handle; tests assert no crash.
- Unknown provider / Anthropic → `ValueError` from factory (legible message).
- Local endpoint unreachable → provider call raises; **resolver does not** (resolver only
  builds, it does not call `embed`). Callers (consumer aspects) own runtime-call try/except.
- `dimension` must reflect the **actual** model output, never a hardcoded 1536 (load-bearing
  for the provider/dim-aware storage in the next aspect).
- Multi-tenancy: resolver is always `org_id`-scoped via `OrgAIConfig` + BYOK.

## 8. Agent execution notes
- **Single agent, sequential phases** — this is one cohesive module; the phases are small and
  ordered, so subagent fan-out adds little. (The two *consumer* aspects are where parallel
  agents pay off.) Still execute via the agents team per the pipeline: dispatch one
  implementation agent for this aspect that runs the 5 TDD phases and reports back per phase.
- **Checkpoint:** after Phase 5, human review of the resolver's degrade matrix (the `None`
  cases are the contract the whole feature leans on).
- **Ask before proceeding if:** the Google SDK is absent AND adding it bloats the image
  unexpectedly — flag rather than silently pin a heavy dep.
- **Do not** modify `template_matcher.py`, `template_saver.py`, `copilot_ws.py`, `main.py`, or
  any migration here — those belong to the consumer aspects; keep this PR purely additive.
