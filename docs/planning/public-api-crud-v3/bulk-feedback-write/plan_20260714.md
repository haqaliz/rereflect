### Phase 2 — Request/response schemas
**Goal:** Pydantic contract with validation-only behavior (no handler yet).
**Files:** `src/api/routes/public_api.py` (or a new `src/schemas/public_bulk.py` if the router prefers) —
keep next to `PublicFeedbackUpdate`.
**Add:**
```python
class PublicFeedbackBulkUpdate(BaseModel):
    model_config = {"extra": "forbid"}
    ids: list[int] = Field(..., min_length=1, max_length=500)
    patch: PublicFeedbackUpdate

class PublicFeedbackBulkResultItem(BaseModel):
    id: int
    status: Literal["updated", "noop", "skipped", "error"]
    reason: Optional[str] = None

class PublicFeedbackBulkResponse(BaseModel):
    matched: int      # unique ids that exist in this org
    updated: int      # results with status == "updated"
    skipped: int      # results with status == "skipped"
    results: list[PublicFeedbackBulkResultItem]
```
**Steps (TDD):**
1. RED: schema tests — `ids=[]` → ValidationError (min_length); `ids` of 501 → ValidationError (max_length);
   unknown top-level field → ValidationError (`extra="forbid"`); `patch` with a bad tag (>50 chars) →
   ValidationError (inherited from `PublicFeedbackUpdate`).
2. GREEN: add the models.
**Validation:** `pytest tests/test_public_api_bulk_feedback.py -v -k schema`.
**Commit:** `feat(public-bulk): request/response schemas for POST /feedback/bulk`

### Phase 3 — Handler: happy path (batched status + per-item results)
**Goal:** Working endpoint for the all-valid case.
**Files:** `src/api/routes/public_api.py` (handler `public_bulk_update_feedback`, registered **before** the
parametric `/feedback/{feedback_id}` routes).
**Handler algorithm:**
1. `require_scope("write")` dependency on the route.
2. Reject empty patch: `if not (data.patch.model_fields_set & {"workflow_status","correction","tags","is_urgent"}): 400 "Nothing to update"`.
3. **Dedupe** `data.ids` preserving first-seen order → `unique_ids`.
4. Query matched: `db.query(FeedbackItem).filter(FeedbackItem.id.in_(unique_ids), FeedbackItem.organization_id == auth.organization_id).all()`; build `by_id = {fb.id: fb}`. `matched = len(by_id)`.
5. Build `results` in `unique_ids` order; ids not in `by_id` → `{status:"skipped", reason:"not_found"}`.
6. **Batched status:** if `patch.workflow_status is not None`:
   `changed = apply_status_change(db, list(by_id.values()), patch.workflow_status, organization_id=..., actor_id=None, actor_label=key_label, resolution_note=patch.resolution_note)`.
   Track `changed_ids = {fb.id for fb,_ in changed}`. **Do not commit yet.**
7. **Per-item fields (non-contagious):** for each matched `fb` (in unique order), open `db.begin_nested()`
   (SAVEPOINT); apply tags / is_urgent / correction exactly as the single PATCH does but calling
   `create_ai_correction(..., commit=False)`; record whether this item changed any field. On exception →
   rollback the savepoint, mark that id `status:"error", reason:<safe msg>`; continue. Track `field_changed_ids`.
8. `db.commit()` **once** (persists status + all per-item field changes + corrections together).
9. Compute per-item status: `error` (already set) > `updated` if id in `changed_ids ∪ field_changed_ids` >
   `noop` (matched, nothing changed).
10. Best-effort side effects **once** (each in try/except, mirror single PATCH): `cache_invalidate` dashboard
    + analytics if anything changed; `dispatch_status_webhooks(db, org, changed, patch.workflow_status, key_label)`
    if `changed`; `await emit_event("workflow:status_changed", {"feedback_ids": sorted(changed_ids), "new_status": patch.workflow_status})` if `changed_ids`.
11. Return `PublicFeedbackBulkResponse(matched, updated=count(updated), skipped=count(skipped), results)`.
**Steps (TDD):**
1. RED: 3 valid ids, `patch={workflow_status:"resolved", is_urgent:true}` → `matched=3, updated=3`, each
   result `updated`; DB reflects new status/urgency; one `workflow:status_changed` event with 3 ids.
2. GREEN: implement.
**Validation:** `pytest tests/test_public_api_bulk_feedback.py -v -k happy`.
**Commit:** `feat(public-bulk): POST /feedback/bulk handler (batched status + per-item results)`

### Phase 4 — Edge cases: skipped / noop / dedupe / cap / non-contagion / scope
**Goal:** Lock every acceptance criterion.
**Steps (TDD) — one test each, RED then GREEN (most already pass after Phase 3; add code only where needed):**
1. **Scope:** no `write` scope → 403; bad/revoked key → 401.
2. **Cross-org / non-existent id** → `skipped/not_found`, not counted in `updated`; the org's own ids in the
   same batch still succeed.
3. **`[good, good, bad]`** → `matched=2, updated=2, skipped=1`; the 2 goods **persisted** (re-query); 3 results.
4. **noop:** id already at target status, no other field → `status:"noop"`; no webhook/event for it.
5. **dedupe:** `ids=[7,7,7]` (exists) → one result, `matched=1`.
6. **cap:** 501 ids → 422 (from schema); nothing mutated.
7. **empty patch:** `patch={}` (or only `resolution_note`) → 400 "Nothing to update".
8. **non-contagion:** force a per-item error (monkeypatch `create_ai_correction` to raise for one id) →
   that id `error`, others `updated` and **persisted**; final response consistent; savepoint rollback verified.
**Validation:** `pytest tests/test_public_api_bulk_feedback.py -v`.
**Commit:** `test(public-bulk): edge cases — skipped/noop/dedupe/cap/non-contagion/scope`

## 6. Edge cases & error handling

- Org scoping on every query (`organization_id == auth.organization_id`); cross-org id is indistinguishable
  from missing → `skipped/not_found` (no 403/404 leak, consistent with the single PATCH 404 behavior).
- `apply_status_change` no-op-skips same status → those ids are `noop` unless a field also changed.
- Per-item `db.begin_nested()` ⇒ a savepoint failure rolls back only that item. Confirm the outer
  transaction is still usable after a nested rollback (SQLAlchemy: yes, if you `except` around the nested block).
- HTTP codes: 200 (results, even with per-item errors), 400 (nothing to update), 401 (auth), 403 (scope),
  422 (schema: empty/over-cap/bad field).
