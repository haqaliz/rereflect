## Phase 1 — Apply writer with concurrency guard (shared)

**Goal:** the single point that changes `workflow_status` + emits ONE timeline event, race-safe.
**File:** in `src/tasks/zendesk_status_sync.py` (or a small `src/services/zendesk_status_apply.py` if cleaner):

```python
def apply_zendesk_status_worker(db, feedback, new_status, *, organization_id, ticket_id, zendesk_status) -> bool:
    old = feedback.workflow_status
    if old == new_status:
        return False
    # Concurrency guard (critique gap #2): conditional UPDATE — only the winner writes the event.
    rows = db.query(FeedbackItem).filter(
        FeedbackItem.id == feedback.id,
        FeedbackItem.organization_id == organization_id,
        FeedbackItem.workflow_status == old,
    ).update({FeedbackItem.workflow_status: new_status}, synchronize_session=False)
    if rows == 0:
        return False   # someone else moved it — no duplicate event
    db.add(FeedbackWorkflowEvent(
        feedback_id=feedback.id, event_type="status_changed", actor_id=None,
        old_value=old, new_value=new_status,
        metadata_={"source": "zendesk", "zendesk_status": zendesk_status, "zendesk_ticket_id": str(ticket_id)},
    ))
    return True
```
**Tests (RED):** apply changes + one event; no-op when already equal; **guarded**: when a concurrent update already moved the row (simulate by pre-updating), `rows==0` → returns False, no event.
**Commit:** `feat(zendesk-status-sync): race-safe worker apply writer (conditional update + source-tagged event)`

```python
def reconcile_feedback(db, feedback, fetched_status, integ) -> str:
    row = db.get(FeedbackZendeskSync, feedback.id)
    stored = row.last_ticket_status if row else None
    action = decide_update(fetched_status, stored)
    if action == "noop":
        return "noop"

    return action
```
