## Phase 1 тАФ `ZendeskClient.show_many`

**Goal:** batch-fetch current ticket statuses by id.
**File to modify:** `services/worker-service/src/clients/zendesk.py`

**Add method:**
```python
_SHOW_MANY_CHUNK = 100  # Zendesk cap

def show_many(self, ids: list) -> dict:
    """Return {str(ticket_id): status} for the given ids. Absent ids omitted."""
    out = {}
    ids = [str(i) for i in ids]
    for i in range(0, len(ids), self._SHOW_MANY_CHUNK):
        chunk = ids[i:i + self._SHOW_MANY_CHUNK]
        if not chunk:
            continue
        resp = self._client.get(f"{self._base}/tickets/show_many.json",
                                params={"ids": ",".join(chunk)})
        data = self._handle_response(resp)      # reuses 429/Retry-After/401/5xx taxonomy
        for t in data.get("tickets", []):
            out[str(t["id"])] = t.get("status")
    return out
```
- Empty `ids` тЖТ no HTTP, `{}`.
- Reuse existing `_handle_response` (429тЖТsleep(Retry-After)тЖТ`ZendeskTransientError`; 5xxтЖТtransient; 401/403тЖТ`ZendeskAuthError`; 404тЖТ`ZendeskNotFoundError` тАФ but `show_many` returns 200 with a subset, so absent ids just don't appear).
