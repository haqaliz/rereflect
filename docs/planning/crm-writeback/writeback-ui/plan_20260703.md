- Manual: `./start-all.sh`, connect HubSpot, create a `number` property in HubSpot, enter its name, Validate, toggle on, confirm status; change a customer's feedback to move their score and confirm the property updates in HubSpot.

## 5. Environment & secrets

- None new (frontend talks to backend; backend holds the token).

## 6. Edge cases & error handling

- Optimistic vs. confirmed toggle: keep the switch controlled by refetched status, not local optimistic state, so a failed enable doesn't visually stick.
- Empty field name while enabling ‚Üí block client-side too (don't rely only on the 422).
- `last_writeback_status` values map to friendly copy (`error: missing_write_scope` ‚Üí "HubSpot token is missing write permission").
- Disconnected/absent status ‚Üí hide the whole card.
