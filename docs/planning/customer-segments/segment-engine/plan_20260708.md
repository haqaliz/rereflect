## Phase 1 — Segment classifier (pure function, TDD) — backend-api

- **Goal:** `classify_segment(...)` returns the correct slug for every rule, tiebreak, and the no-usage path. Pure, no DB.
- **Prerequisites:** none.
- **Files:**
  - CREATE `services/backend-api/src/services/segment_service.py`
  - CREATE `services/backend-api/tests/test_segment_service.py`
- **Design (implement to make tests pass):**
  - Canonical slug set + priority order in ONE structure:
    ```python
    SEGMENT_AT_RISK = "at_risk"; SEGMENT_SILENT_CHURNER = "silent_churner"
    SEGMENT_DORMANT = "dormant"; SEGMENT_POWER_USER = "power_user"
    SEGMENT_HAPPY_ADVOCATE = "happy_advocate"; SEGMENT_NEW = "new"
    SEGMENT_UNSEGMENTED = "unsegmented"
    SEGMENT_SLUGS = [ ...6 in priority order..., SEGMENT_UNSEGMENTED ]  # exported; API imports this
    ```
  - `classify_segment(*, health_score, risk_level, churn_probability, usage, sentiment_direction, last_feedback_at, created_at, now) -> str`
    where `usage` is a small dataclass/None (fields: `last_active_at, active_days_30d, distinct_feature_count, usage_score, first_seen_at`). Pure — caller passes primitives, not ORM rows, so tests need no DB.
  - Evaluate top-down, first match wins (thresholds reuse `usage_score_service` constants — import them):
    1. `at_risk`: `risk_level in {"at_risk","critical"}` OR (`churn_probability is not None and churn_probability >= 0.5`).
    2. `silent_churner`: **usage present** AND declining engagement (`usage.active_days_30d is not None and usage.active_days_30d < FREQUENCY_LOW_MOD_DAYS`) AND `sentiment_direction == "declining"` AND (`last_feedback_at is None or last_feedback_at < now - 30d`).
    3. `dormant`: (**usage present** AND `usage.last_active_at < now - RECENCY_AGING_DAYS`) OR (**usage absent** AND `last_feedback_at is not None and last_feedback_at < now - 60d`).
    4. `power_user`: **usage present** AND `usage.usage_score is not None and usage.usage_score >= 75` AND `usage.active_days_30d is not None and usage.active_days_30d >= FREQUENCY_MOD_HIGH_DAYS`.
    5. `happy_advocate`: `health_score >= 75` AND `sentiment_direction in {"improving","stable"}`.
    6. `new`: `created_at is not None and created_at >= now - 14d` AND `feedback_count` low (pass `feedback_count`; `< 3`).
    7. else `unsegmented`.
  - **Usage-gating invariant (R4):** rules 2, 4, and the first arm of 3 must early-skip when `usage is None`.
- **Tests (RED first):** truth-table — one row per segment hitting exactly that rule; priority tiebreak (a power-user row that is also `at_risk` → `at_risk`; an advocate who is also `dormant` by recency → `dormant`); **no-usage path** (usage=None → never `power_user`/`silent_churner`; not falsely `dormant` unless the feedback-recency arm applies); `unsegmented` fallback; boundary values at each threshold.
- **Validation:** `cd services/backend-api && pytest tests/test_segment_service.py -v` green.
- **Commit:** `feat(segments): rule-based segment classifier (pure, TDD)`

## Testing strategy

- **Unit (Phase 1):** pure classifier truth table — the bulk of correctness lives here; no DB.
- **Integration (Phase 3):** on-ingest assignment via `update_customer_health`.
- **Task (Phase 4):** recompute over seeded rows, staleness reclassification, archived-skip, coverage.
- **Regression:** existing customer-health/usage suites stay green (segment write is additive). Scope test
  runs to the touched areas per repo test gotchas (avoid the known full-suite segfault in
  `test_report_ws.py`; don't trust a bare full-suite green).
- **Commands:** `cd services/backend-api && pytest tests/test_segment_service.py tests/test_segment_on_ingest.py -v`; `cd services/worker-service && pytest tests/test_recompute_segments.py -v`.

## Edge cases & error handling

- **No `CustomerUsage` row** (usage not wired — common self-host default): usage-gated rules skip; no false `dormant`. (Explicit test.)
- **Null `last_active_at` / `churn_probability` / `created_at`:** guarded (`is not None`) before comparison.
- **`compute_sentiment_trend` returns `stable` on no data** (verified :590) → advocate rule needs `health_score>=75` too, so no-feedback customers aren't blanket-advocates.
- **Segment write failure** must never break health upsert (try/except + log; segment left unchanged).
- **Multi-tenancy:** classifier is row-scoped/org-agnostic; all queries already org-scoped. No new cross-org surface in this aspect.
