## Contract from segment-api (already shipped)

- List: `GET /api/v1/customers/?segment=<slug>` filters; each row + profile carry `segment` (nullable).
- Slugs: `at_risk, silent_churner, dormant, power_user, happy_advocate, new, unsegmented`; `null` = not
  computed. Invalid `?segment=` → 422.

## Edge cases

- `segment` null/undefined → render the neutral/"—" treatment, never crash.
- Filter value "all" clears the param (don't send `segment=all`).
- Theme: verify chips legible in light AND dark (tokens already theme-aware; don't hardcode).

## Agent notes

- Import the slug→label map from ONE place (SegmentBadge meta) so the filter dropdown and the badge can't
  drift, and so it matches the backend `SEGMENT_SLUGS` order.
- Do NOT re-fetch or recompute segment client-side — it's a plain field on the row/profile.
- Keep components small (CLAUDE.md). Use CSS vars only.
