## Phase 2 — Pure reconcile core (no I/O)  ⟂ parallel with Phase 3

- **Goal:** the deterministic decision function, fully unit-tested with zero I/O — the piece
  Zendesk/Asana reuse later.
- **Prereq:** none (pure). Can run parallel to Phase 3.
- **Files:**
  - `services/backend-api/src/services/status_sync_core.py` (new)
  - `services/backend-api/tests/test_status_sync_core.py` (new)
- **API to implement (pure functions):**
  ```python
  DEFAULT_CATEGORY_MAP = {"done": "resolved", "indeterminate": "in_review", "new": "new"}
  CATEGORY_RANK = {"new": 0, "indeterminate": 1, "done": 2}  # most-advanced wins

  def resolve_target_status(category: str, mapping: dict | None) -> str | None:
      """mapping (JSON) overrides DEFAULT_CATEGORY_MAP per key; unknown category -> None."""

  def is_seed(stored_category: str | None) -> bool:
      """True when stored is None -> first observation -> seed, do NOT apply."""

  def most_advanced(categories: list[str]) -> str | None:
      """highest CATEGORY_RANK across a feedback's live links."""

  def decide_link_update(fetched_category, fetched_name, stored_category):
      """-> ('seed'|'changed'|'noop', fetched_name, fetched_category). 'changed' only when
         stored is not None and fetched_category != stored_category."""
  ```
- **Steps:** RED tests first for: default map, JSON override, unknown category → None, seed
  (None→value) classified `seed` not `changed`, same-value → `noop`, transition → `changed`,
  most-advanced tie-breaking, invalid target guarded against `VALID_STATUSES`
  (`new/in_review/resolved/closed`).
- **Validation:** `pytest tests/test_status_sync_core.py -v` (pure, fast, no DB).
- **Commit:** `feat(jira-status-sync): pure reconcile core (category map, seed, most-advanced)`

    `test_jira_sync_task.py` (new)
- **Steps:**
  1. **Client** (`clients/jira.py`): httpx Basic auth `(email, token)`, base `{site_url}/rest/api/3`,
     15s timeout, `_handle_response` translating 429(`Retry-After` sleep)→`JiraTransientError`,
     5xx→transient, 401/403→`JiraAuthError`, 404→`JiraNotFoundError`; `search_issues(keys)` (mirror
     Phase 3). Never log token. RED test the throttle mapping (mirror `clients/zendesk.py:102-125`).
  2. **Task** (`tasks/jira_sync.py`), mirror `zendesk_sync.py`:
     - `sync_all_jira` fan-out: query `JiraIntegration.is_active AND status_sync_enabled` →
       `sync_jira_org.delay(id)` each, per-org try/except.
     - `sync_jira_org(self, integration_id)` `@shared_task(bind=True, max_retries=3,
       default_retry_delay=30)` → delegate to `_sync_jira_org_body(integration_id, db, client=None)`
       (extracted for tests). Body: load integ; guard inactive/disabled; decrypt token (local
       `_decrypt`, `LLM_ENCRYPTION_KEY`, missing→`{"status":"error","reason":"missing_encryption_key"}`);
       gather this org's `FeedbackJiraIssue` keys; chunk → `client.search_issues`; for each link call
       `decide_link_update(...)`; **seed** → set link fields only; **changed** → set link fields +
       collect (feedback_id, most_advanced target across that feedback's links); apply via a
       worker-local status writer that mirrors `apply_status_change` semantics (see step 3); set
       `integ.last_synced_at`, `last_sync_status="success"`, `last_error=None`.
     - Errors: `JiraTransientError`→`self.retry`; `JiraAuthError`→record `last_sync_status="error"`,
       no retry, **no disconnect**; durable terminal-status write in a fresh `get_db_session()`
       (mirror `zendesk_sync.py:97-133`).
  3. **Status write from worker:** the worker can't import backend-api's `apply_status_change`.
     Implement a small worker-local `apply_status_change_worker(db, feedback, new_status, *,
     organization_id, actor_label, metadata)` that: no-ops on same value; sets `workflow_status`;
     inserts a `FeedbackWorkflowEvent` row (`event_type="status_changed"`, old/new value, metadata
     `{source:"jira", jira_status, jira_status_category, jira_issue_key}`); leaves outbound webhook
     dispatch to a follow-up (see Agent Notes — worker webhook dispatch parity is a decision point).
  4. **Beat** (`celery_app.py`): add `"src.tasks.jira_sync"` to `include`; add

## Edge Cases & Error Handling
- Disabled/inactive org → zero Jira calls.
- First poll after enable → seed only, **no** status write (safety).
- Same-value poll → no-op (no event/webhook).
- Multi-issue feedback → most-advanced category wins (no regress).
- 429→retry w/ `Retry-After`; 5xx→transient; 401/403→record error, keep connected; 404→skip that
  link; one bad link/org never aborts the batch.
- Bad `status_mapping` (unknown key or non-`VALID_STATUSES` value) → 422 at PATCH.
- Large org → chunk JQL (≤50 keys) + page cap; multi-beat convergence acceptable.
- `organization_id` scoping enforced on every route + every worker query.
