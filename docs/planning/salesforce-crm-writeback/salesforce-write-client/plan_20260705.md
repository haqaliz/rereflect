## Edge cases / error handling
- **Raw number** in the PATCH body — regression-guard against copying HubSpot's stringification.
- SF update returns **204 No Content** on success — treat 2xx (incl. 204) as success.
- Describe can 403 if the connected app/user lacks object access → `missing_write_scope`.
- Validation service returns tuples, never raises (route translates to 400/422).
