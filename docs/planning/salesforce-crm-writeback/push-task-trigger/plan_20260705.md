## Edge cases / error handling
- Re-query fallback must be a bounded SOQL, not a full `list_contacts()` scan.
- Never flip `is_active` in the writeback task (owned by read-sync).
- `abs(diff) < 2` + `score_unchanged` idempotency reused for both providers.
- Daily-limit → `deferred: daily_limit` + retry, not a silent drop.
