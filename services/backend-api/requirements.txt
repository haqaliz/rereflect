# lxml + xmlsec MUST link the SAME libxml2, or import dies with
#   "lxml & xmlsec libxml2 library version mismatch".
# Mixing prebuilt wheels does not guarantee that, so we force BOTH from source (--no-binary below)
# against the system libxml2 that the Dockerfile / host provides.
#
# xmlsec version is platform-split because no single SOURCE version fits both targets:
#   - Debian 12 (bookworm) image  -> libxmlsec1 1.2.37 -> needs xmlsec <= 1.3.13
#   - macOS dev host (brew)        -> libxmlsec1 1.3.x  -> needs xmlsec >= 1.3.14
# The Python API is identical across these; tests run on both. (A future Docker bump to a base with
# libxmlsec1 >= 1.3.0 — e.g. trixie — would let us unify on one version.)
# macOS source build needs brew paths on PKG_CONFIG_PATH/CPPFLAGS/LDFLAGS (see docs/SELF_HOSTING.md dev note).
